无需安装cc switch，10行命令帮你配置Claude Code+deepseek

Claude Code 想必大家都很熟悉了，它是一个运行在终端内的 AI 编程助手。

然而，网上大部分安装教程，都推荐先安装一个工具，那就是 cc switch。

当然，这个可以一键切换多种 AI 编程 CLI 的工具有它的优点所在。

下面，将站在一个**小白**的角度，带大家一步一步配置国内**性价比之王**——DeepSeek 接入 **Agent 工具**——Claude Code。

如果你电脑中 Node.js 的版本为 18+，并且安装了 Git，可以**直接跳转至** [三、安装 Claude Code](#%E4%B8%89%E5%AE%89%E8%A3%85-claude-code)。

一、安装 Node.js
------------

打开 [Node.js 官网](https://nodejs.org/zh-cn/download)，根据你的系统选择：

*   **Windows**：下载 `.msi` 安装包（64-bit 或 32-bit）
*   **macOS**：下载 `.pkg` 安装包
*   **Linux**：使用 NodeSource 或 nvm 安装

![](https://img2024.cnblogs.com/blog/3019595/202605/3019595-20260521233213593-1562610589.png)

双击下载的安装包，一路 Next：

*   双击下载好的 `.msi` 安装包
*   欢迎界面点击 **Next**
*   勾选 "I accept the terms in the License Agreement"，点击 **Next**
*   安装路径保持默认 `C:\Program Files\nodejs\`，点击 **Next**
*   **关键步骤**：在 Custom Setup 界面，确保 "npm package manager" 被选中，点击 **Next**
*   **重要**：在 Tools for Native Modules 界面，勾选 "Automatically install the necessary tools"，点击 **Next**
*   点击 **Install** 开始安装
*   安装完成后点击 **Finish**
*   如果弹出独立窗口安装 Windows Build Tools，按提示继续安装（这会自动安装 Python 和 C++ 编译工具）

安装完成后，按 `Win + R`，输入 `cmd` 打开命令提示符，输入：

    node -v
    

看到出现版本号，就代表安装成功了。

二、Windows 用户安装 Git for Windows
------------------------------

如果你不是 Windows，可以跳过这一步。

打开 [Git 官网](https://git-scm.com/install/windows)，下载会自动开始，如果没开始，手动点击 **64-bit Git for Windows Setup**。

![](https://img2024.cnblogs.com/blog/3019595/202605/3019595-20260521233355049-736340455.png)

运行下载的 `.exe` 安装程序，安装选项建议：

*   双击下载的 `Git-2.x.x-64-bit.exe` 文件
*   遇到安全警告点击 "运行"
*   阅读许可证，点击 **Next**
*   安装路径保持默认 `C:\Program Files\Git`，点击 **Next**
*   组件选择保持默认即可，点击 **Next**
*   开始菜单文件夹保持默认，点击 **Next**
*   **重要**：选择默认编辑器，建议选 "Use Visual Studio Code as Git's default editor"（如果装了 VS Code）或 "Use Nano"，点击 **Next**
*   **关键步骤**：调整 PATH 环境，选择 "Git from the command line and also from 3rd-party software"，点击 **Next**
*   HTTPS 传输后端，选择 "Use the OpenSSL library"，点击 **Next**
*   行结束符转换，选择 "Checkout Windows-style, commit Unix-style line endings"，点击 **Next**
*   终端模拟器，选择 "Use MinTTY"，点击 **Next**
*   其他选项全部保持默认，一路点击 **Next**
*   最后点击 **Install** 安装
*   安装完成后取消勾选 "Launch Git Bash"，点击 **Finish**

安装完成后，重新打开命令提示符，输入：

    git --version
    

看到出现版本号，就代表安装成功了。

三、安装 Claude Code
----------------

### 1\. 切换镜像源

因为国外的 npm 官方源在国内访问会很慢，甚至可能卡住不动。所以，咱们换成国内能用的镜像源来加速。

开始菜单任务栏搜索 **PowerShell**，选择以管理员身份运行：

![](https://img2024.cnblogs.com/blog/3019595/202605/3019595-20260521233525486-1319762280.png)

在 PowerShell 里执行：

    npm config set registry https://registry.npmmirror.com/
    

验证一下：

    npm config get registry
    

确认显示 `https://registry.npmmirror.com/`。

### 2\. 开始安装 Claude Code

还是在上面的 PowerShell 窗口中执行：

    npm install -g @anthropic-ai/claude-code
    

`npm install -g` 里的 `-g` 意思是全局安装，不管当前在哪个目录，装完后，`claude` 命令在所有项目路径下都能用。

等它跑完，然后运行：

    claude --version
    

显示出版本号，即表示安装成功。

四、获取 DeepSeek API
-----------------

### 1\. 调用 API 的前提

打开 [DeepSeek API 官方网站](https://platform.deepseek.com/)，注册并登录账号。

可以先充值 10 元试试，后续再充值。当然，你也可以自定义 1 元，先获取到 API。

### 2\. 创建 API keys

这是**关键**的一步，输入名称后，请你先保存记住这个 Key：`sk-……`。当然，后续忘了也可以再创建新的 API key，删除旧的 API key。

五、配置环境变量
--------

1.  依然是在 PowerShell 下，**依次**执行：

> **注意**：第 2 条命令，你必须替换为自己真实的 API key。

    [Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "https://api.deepseek.com/anthropic", "User")
    

    [Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "你的真实API Key", "User")
    

    [Environment]::SetEnvironmentVariable("ANTHROPIC_MODEL", "deepseek-v4-pro[1m]", "User")
    

    [Environment]::SetEnvironmentVariable("ANTHROPIC_DEFAULT_OPUS_MODEL", "deepseek-v4-pro[1m]", "User")
    

    [Environment]::SetEnvironmentVariable("ANTHROPIC_DEFAULT_SONNET_MODEL", "deepseek-v4-pro[1m]", "User")
    

    [Environment]::SetEnvironmentVariable("ANTHROPIC_DEFAULT_HAIKU_MODEL", "deepseek-v4-flash", "User")
    

    [Environment]::SetEnvironmentVariable("CLAUDE_CODE_SUBAGENT_MODEL", "deepseek-v4-flash", "User")
    

    [Environment]::SetEnvironmentVariable("CLAUDE_CODE_EFFORT_LEVEL", "max", "User")
    

上面的命令行，是把环境变量**永久写入系统**，以后每次打开新项目目录终端时都**自动生效**。

> 注：在 DeepSeek 官网上，配置的 DeepSeek 环境变量是**临时的**。当你打开一个新项目时不会**生效**。环境变量只在设置时的那个窗口有效，关掉就没了。所以必须要在同一个 PowerShell 窗口里重新设置变量，再启动 Claude Code。

2.  下面以表格形式，介绍一下每个环境变量的含义：

环境变量

含义

`ANTHROPIC_BASE_URL`

把请求发到 DeepSeek 的服务器，而不是 Anthropic 官方

`ANTHROPIC_AUTH_TOKEN`

你的 DeepSeek API Key，用于身份验证和计费

`ANTHROPIC_MODEL`

默认使用的模型，`deepseek-v4-pro[1m]` 是 DeepSeek 最新旗舰模型

`ANTHROPIC_DEFAULT_OPUS_MODEL`

原本调用 Claude Opus（最强模型）时的替代模型

`ANTHROPIC_DEFAULT_SONNET_MODEL`

原本调用 Claude Sonnet（平衡模型）时的替代模型

`ANTHROPIC_DEFAULT_HAIKU_MODEL`

原本调用 Claude Haiku（轻量快速）时的替代模型

`CLAUDE_CODE_SUBAGENT_MODEL`

子任务代理使用的模型，用更快的 flash 版节省成本

`CLAUDE_CODE_EFFORT_LEVEL`

模型思考深度，`max` 表示最强推理能力

3.  验证是否成功：

**关闭当前**的 PowerShell 窗口，**重新**打开一个新的，运行：

    $env:ANTHROPIC_BASE_URL
    

如果输出 `https://api.deepseek.com/anthropic`，说明永久生效了。

4.  更新永久环境变量：

如果你旧的 API key 不用了，可以这样更改为新的——只需重写那一条变量，在 PowerShell 里执行：

    [Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "新的API Key", "User")
    

**更新其他变量同理**，比如以后**想换模型**，执行：

    [Environment]::SetEnvironmentVariable("ANTHROPIC_MODEL", "新模型名", "User")
    

同样**新开窗口**生效。

六、开始使用 Claude Code
------------------

1.  打开你的项目文件夹
2.  鼠标在文件夹内空白处右键

![](https://img2024.cnblogs.com/blog/3019595/202605/3019595-20260521233640761-787268400.png)

选择在终端打开，或者直接 `cmd`：

![](https://img2024.cnblogs.com/blog/3019595/202605/3019595-20260521233717871-167884952.png)

3.  输入指令：`claude`
    
4.  敲回车即可以开始对话了
    
5.  第一次运行指令 `claude` 时，会出现以下情况：
    

**情况 1：**

     Let's get started.
     Choose the text style that looks best with your terminal
     To change this later, run /theme
       1. Auto (match terminal)
     > 2. Dark mode ✓
       3. Light mode
       4. Dark mode (colorblind-friendly)
       5. Light mode (colorblind-friendly)
       6. Dark mode (ANSI colors only)
       7. Light mode (ANSI colors only)
    

现在你需要做一个初始选择：它问你要用什么主题。默认选中的是 **2\. Dark mode**（暗色模式），直接按**回车**即可。也可以按数字键选其他主题。选完后你就可以开始使用了。

比如可以试试输入：

> 帮我创建一个 hello world 的 Python 脚本

或者任何你想做的编程任务。

**情况 2：**

     Claude Code'll be able to read, edit, and execute files here.
     Security guide
     > 1. Yes, I trust this folder
       2. No, exit
    

这是安全确认提示，问你是否信任当前所在的目录。Claude Code 会拥有读取、编辑和执行这个目录下所有文件的权限。
