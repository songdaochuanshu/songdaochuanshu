龙虾白嫖指南，请查收~
===========

![龙虾白嫖指南，请查收~](https://img2024.cnblogs.com/blog/602535/202603/602535-20260325172524597-2012163136.png) 本文主要介绍了基于飞书OpenClaw，配置ModelScope的开源模型，解决飞书OpenClaw的token受限的情况。

**不说废话，直接开干~**

1\. 飞书注册
========

*   打开地址: [https://openclaw.feishu.cn/](https://openclaw.feishu.cn/)
*   点击"立即部署"  
    
*   输入手机号登录 -> 验证码 -> 开始使用

2\. 创建飞书OpenClaw
================

*   点击"创建"，并勾选"已阅读并同意服务规则"!  
    
*   创建OpenClaw项目: 设置项目名称+配置飞书渠道，自己命名即可，然后单击"创建" ，等待1分钟。  
    

3\. 电脑或手机安装飞书应用
===============

*   进入飞书手机端或电脑端，以手机号进行注册，即可在消息也看到刚刚创建的龙虾，然后就可以直接体验了。  
    

4\. 白嫖进阶
========

*   飞书目前提供的openclaw的使用额度很有限，多问几个问题就没有额度了。那是否有更多的体验额度可以使用呢？当然！
*   魔搭社区通过API-Inference为每位注册用户提供了每天总数为2000次的调用额度，完全满足日常工作需要。API-Inference将开源模型服务化并通过API接口进行标准化，让开发者和用户可以更轻量和迅捷的方式体验开源模型。
*   可体验的开源模型总计超过4.4W，包括目前火遍海内外的MiniMax M2.5, Kimi 2.5, Qwen全家桶等等均可免费体验。

4.1 注册ModelScope并绑定阿里云账号
------------------------

*   [注册账号](https://modelscope.cn/docs/accounts/registration)
*   [绑定阿里云账号，并实名认证](https://modelscope.cn/docs/accounts/aliyun-binding-and-authorization)

4.2 创建访问令牌
----------

*   访问令牌类似于各模型厂商的apikey。[点击查看具体创建方式](https://modelscope.cn/docs/accounts/token)。

4.3 飞书OpenClaw设置ModelScope的访问模型
-------------------------------

*   拿到了令牌后，即可在飞书OpenClaw中进行更换模型。点击飞书聊天窗口下方的"管理"，即可跳转到网页查看OpenClaw的管理页面。  
    
*   点击控制台邮编的"代码</>"按钮，然后openclaw.json进行编辑，其中apiKey就是上面申请的token。

    # 1. models.providers与"miaoda"同级写入"modelscope"的如下内容
    "modelscope": {
    	"baseUrl": "https://api-inference.modelscope.cn/v1",
    	"apiKey": "ms-xxxxxxxx",
    	"api": "openai-completions",
    	"models": [
    		{
    			"id": "moonshotai/Kimi-K2.5",
    			"name": "Kimi-K2.5",
    			"reasoning": false,
    			"input": [
    				"text",
    				"image"
    			],
    			"contextWindow": 200000,
    			"maxTokens": 16384
    		},
    		{
    			"id": "Qwen/Qwen3.5-35B-A3B",
    			"name": "Qwen3.5-35B-A3B",
    			"input": [
    				"text",
    				"image"
    			]
    		},
    		{
    			"id": "MiniMax/MiniMax-M2.5",
    			"name": "MiniMax-M2.5",
    			"input": [
    				"text"
    			]
    		}
    	]
    }
    
    # 2. agents.defaults进行修改
    # (1) model文本对话模型主要是Kimi-K2.5，辅助模型是MiniMax-M2.5
    # (2) imageModel图片主模型是Kimi-K2.5，辅助图片模型是Qwen3.5-35B-A3B
    # (3) models模型列表添加
    ......
    "verboseDefault": "off",
    "model": {
    	"primary": "modelscope/moonshotai/Kimi-K2.5",
    	"fallbacks": [
    		"modelscope/MiniMax/MiniMax-M2.5"
    	]
    },
    "imageModel": {
    	"primary": "modelscope/moonshotai/Kimi-K2.5",
    	"fallbacks": [
    		"modelscope/Qwen/Qwen3.5-35B-A3B"
    	]
    },
    "models": {
    	"miaoda/miaoda-model-auto": {
    		"alias": "Miaoda Auto"
    	},
    	"miaoda/miaoda-auto-multimodal": {
    		"alias": "Miaoda Multimodal"
    	},
    	"miaoda/miaoda-model-flash": {
    		"alias": "Miaoda Flash"
    	},
    	"modelscope/moonshotai/Kimi-K2.5": {
    		"alias": "Kimi-K2.5"
    	},
    	"modelscope/Qwen/Qwen3.5-35B-A3B": {
    		"alias": "Qwen3.5-35B-A3B"
    	},
    	"modelscope/MiniMax/MiniMax-M2.5": {
    		"alias": "MiniMax-M2.5"
    	}
    },
    ......
    
    

4.4 配置生效
--------

*   切换至控制台，打开"聊天页面"，然后输入"重启openclaw的网关gateway"
*   当前页面会关闭，等待1分钟后再进行链接，然后可以查询当前的主要模型
*   之后就可以在电脑端或手机端进行愉快的聊天啦~~

5\. 效果验证
========

*   图片生成: 默认使用miaoda-studio-cli text-to-image生成的  
    
*   图片分析: 默认使用kimi 2.5分析  
    
*   验证模型是否生效
