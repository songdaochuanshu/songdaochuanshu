通过.ovf和.vmdk导入VMware虚拟机
=======================

背景介绍：跟友人咨询有没有免费的EDA软件可以玩玩，发来一个百度网盘链接

好家伙，压缩包131G！笔记本电脑单个磁盘分区都不够放的，更别说解压缩了。心一横，800多块大洋入手了一款PSSD——闪迪E30，1T。

操作步骤：

1、安装VMware 17.6

2、将解压缩后的.vmdk以及.ovf、.mf放在同一个子目

3、导入

等待时间较长

第一次尝试：首次尝试是想着以后用着方便，且希望性能会好一些，因此计划把虚拟机导入到电脑磁盘中。磁盘分区合并凑了250G，压缩包放在PSSD，解压缩到电脑磁盘里。此时，除了待导入文件，剩余120G左右，我琢磨着肯定够了吧？

Failed to write content to disk F:\\RedHatPublic\\\\RedHat7d9-disk1.vmdk. Reason: There is not enough space on the file system for the selected operation。

第二次尝试：解压缩文件放在移动固态中，虚拟机导入到电脑磁盘中，此时剩余空间250G总该够了吧？！结果还是一样的报错，空间不足。

第三次尝试：解压缩文件文件放在PSSD中，虚拟机也导入到PSSD中，不想出了新问题：SHA digest of file RedHat7-disk1.vmdk does not match manifest。后面才了解到.mf文件中存放了.ovf和.vmdk的SHA256值，cmd中计算了下，前者匹配，后者不一致。

第四次尝试：更新.mf文件，再次导入，终于成功。
