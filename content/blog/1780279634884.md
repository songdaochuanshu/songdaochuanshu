花了一晚上AI Coding， 在不熟悉的领域，使用AI帮同事解决了跳槽的小问题
========================================

1\. 背景
======

前几天玩的好的同事悄悄私聊我说：有没有一些AI相关的信息、项目进行推荐的。我回答说我看的也比较散，GitHub社区看一点，OpenAI社区看一点，看的很散。

他说他准备跳槽了(受不了老板了)，想学习一下AI相关的知识，做一下AI相关的项目。

我想了想的确，现在AI发展的这么快，每天/每周都会出一些新的概念，根本来不及了解学习，而自己获取信息又比较散，因此周末趁着Github Copilot 6.1号政策修改之前，花了一晚上写了一个 【每日AI前沿信息】。

我这次采用的是VUE前端 + Python后端，都不是我熟悉的领域（我自己是Java Boy），来判断VB Coding是否能让我在一个完全不熟悉的领域解决一个特定的问题。

2\. 功能点
=======

说干就干，感叹于AI的发展，在我自己不熟悉的前端VUE + 后端语言Python情况下，我仅仅花了一个晚上开发 + 一个白天优化细节，就全部开发完成进行上线。 （到底是好事还是坏事呢）

------------

首页可以看到8个最新AI网站的资讯，包括 GitHub Trending、Hacker News、V2EX 、TLDR AI、OpenAI、Anthropic、InfoQ AI Development。

其中最让我满意的是 我引入了GPT-4 来进行总结，而不是直接获取原文，让GPT来帮我总结提取最精炼的给到我，并且翻译为中文。

那么我每天早上花5-10分钟快速浏览一下，哪些我感兴趣点进去详细了解一下，哪些不感兴趣直接忽略。

3\. 对AI Coding的一点感想
===================

我应该是最积极、最早拥抱AI Coding的一批人，从22年使用Github Copilot，到23年Cursor，再到24年的Claude Code / Codex。期间使用AI做了3-5个项目，当然都是个人项目

我个人发现AI的能力边界真的非常的大，尤其是从0-1的项目，基本就是完全有手就行，根本不用思考怎么实现。更多的是需要产品经理、设计、项目管理的能力。

也就是编码再也不是一个问题了，而更多的是我的这个产品怎么做、怎么交互、页面怎么设计、项目怎么管理，这才是一个产品的核心竞争力。而我们以前传统程序员的特有的编码能力被 AI 极大地磨平了，对产品经理来说是重大的利好，对程序员来说可能真的是一个灾难。

当然对于公司工业级别的项目，AI还是差点意思。因为他会乱写，并不符合我们的要求，所以出现了很多工程去解决这个问题，比如‌**Harness Engineer**‌等等，但是在未来1-3年之内，我感觉这个大概率不再是问题。

4\. 总结
======

在不熟悉的领域，使用AI可以帮你快速“做成”一件事情，因为没有AI不熟悉的领域，它可以辅导你完成99%的问题。

也就是虽然AI对程序员有影响，但是他把整个编程领域给扩大了，比如我看到有60岁的大爷使用workbuddy在分析股票(雪球老哥)，我看到有产品经理(我司+脉脉很多老哥)使用cursor直接把demo做出来了，我甚至看到有明星(胡彦斌)在Coding

也就是本来1000W人的市场，有了AI整个市场得规模可能会扩大10倍，直接到1亿人开始AI Coding，那规模不可想象

未来我猜测，熟悉的老手+AI 薪资会越来越高，越来越抢手。而新手(刚入门的工程师)，可能会被慢慢淘汰。就像当时纺织机的出现，一大批纺织工人失业，但是会操作纺织机的工人却非常抢手。我们公司最近1年都在清退外包，然后鼓励大家使用AI
