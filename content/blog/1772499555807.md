这个年轻的开源项目，想让每个人都能拥有自己的专业级 AI 智能体
================================

刚刚过去的 2025 是 AI 智能体爆发的一年，别再只把 AI 当聊天机器人了！

正是从这一年，AI 才开始真的可以处理工作中的日常任务，但无论对于团队还是个人，搭建 Agent 往往只有技术人员才能胜任。而一个 AI 智能体背后需要掌握 LLM、RAG、LangChain、MCP、前后端开发等多个领域的知识，想想就让人望而却步。

今天 HelloGitHub 就分享一款部署简单、一行代码也不用写，仅靠自然语言就能轻松构建出来智能体的开源项目——Nexent。

一、Nexent 是什么？
-------------

Nexent 是一个开源的零代码智能体自动生成平台——无需编排、复杂的拖放操作，使用时 MCP、RAG 这些都不需要考虑。**全程无需写代码，技术小白也可以轻松构建一个属于自己的智能体**。不管是撰写深度研究报告、分析表格、设计海报...这些智能体都可以通过 Nexent 轻松配置。

> GitHub 地址：[github.com/ModelEngine-Group/nexent](https://github.com/ModelEngine-Group/nexent)

相较于其它开源智能体平台，Nexent 不仅开源而且采用 MIT 协议。这意味着个人或团队都可以随便把代码拉下来学习和二次定制开发，同时 **Nexent 维护积极，几乎每周都有更新**。

二、快速上手
------

Nexent 的部署非常简单，只需要 2C6G 的配置就可以通过 Docker 一键轻松部署，实测只需四行命令，5 分钟左右即可运行（耗时主要取决于镜像下载速度）。

    git clone https://github.com/ModelEngine-Group/nexent.git
    cd nexent/docker
    cp .env.example .env # fill only necessary configs
    bash deploy.sh
    

初看下来 Nexent 没那么多花里胡哨的功能，重点放在实用上。现在大模型越来越强，用自然语言来把智能体“说清楚”其实是更直接，也更好用的方式，这恰恰也是 Nexent 的核心功能。

**核心功能**

*   **智能体自动生成**：全程无需写代码，只需要在页面上配置即可。Nexent 内置了提示词生成能力，用户只需要自然语言描述需求，就能自动根据用户的需求生成高质量提示词。
*   **多智能体自主业务决策**：默认提供了基于 ReAct 框架的智能体运行时。实现智能体自动进行思考、任务规划、决策和执行以及调用相关的工具集和 MCP，从而高效理解和完成用户下达的指令。
*   **默认知识库集成**：支持多种知识来源，用户只需要上传自己的文档，Nexent 会自动完成切片、向量化等复杂逻辑。
*   **多模态对话**：知识库可以包含图像，对话过程中也可以使用文本、图像、音频等多种数据类型的输入/输出。
*   **智能体共享**：不仅可以尝试自己构建智能体，还可以从智能体市场下载其他人分享的智能体，真正做到开箱即用。

三、使用案例
------

既然在本地部署完了 Nexent，那就上手一试试吧！

所以我用 HelloGitHub 月刊内容为例体验了一圈其功能，不得不说 Nexent 确实有点东西。

### 3.1 自动构建知识库

用 Nexent 构建一个知识库非常简单，只需要把本地的文档拖进去即可快速构建一个知识库，支持 docx、ppt、Markdown、pdf 等多种常见格式，系统会自动计算分片策略，进行分段并落库。

我把《HelloGithub 月刊》的 20 期文件批量上传到 Nexent，几秒钟就帮我构建好了一个 HelloGithub 知识库。

### 3.2 智能生成知识库

Nexent 相较于其他开源方案不同的地方在于创新性地增加了知识库自动总结功能，知识库创建成功之后会通过知识库的内容智能生成该知识库的概览。**这些概览会在 AI Agent 检索时提供知识库的全景视图，从而提升在多知识库索引时的速度和准确度**。

### 3.3 智能体快速配置

有了知识库之后，我们就可以通过快速配置直接创建一个 Agent 智能体。由于 Nexent 已经在平台层封装了主流的包括大模型、语音模型、向量模型在内的多个 AI Agent 开发必备的基础技术组件。

因此只需要根据说明即可完成 AI Agent 所需的基础能力的配置，之后在创建 AI Agent 时可以关联上面创建的知识库，一个 AI Agent 还可以关联多个知识库。

### 3.4 创建智能体

在 Nexent 中创建智能体的过程也非常丝滑，它默认提供了十多种常用的内置工具以及开箱即用的 MCP 集成方式，只需要输入在线 MCP 的链接即可直接添加 MCP，系统就会自动解析该 MCP 的 Tools。

同时对于一些需要本地运行的 MCP，Nexent 的 MCP 工具也提供了非常方便的 MCP 运行环境。接入之后我们可以手动选择把哪些 MCP Tool 集成到我们的 Agent 中。

工具和 MCP Tools 选择好之后，只需一句话描述应用场景，点击生成系统便能自动构建专业的提示词。之后点击保存就能够得到一个可以调用多种工具，还会自主思考规划任务的 AI Agent 了。

至此，新建的智能体已经通过接入 HelloGithub 以往的内容作为知识库，并接入了联网搜索和网页分析这两个 MCP 工具，生成了一个 HelloGithub 选题智能体。

### 3.5 图文智能体

作为一个开源博主，文章封面也是每次让我很头疼的事情，因此我想着也许可以再做一个封面设计智能体帮我顺便把封面生成了。

多智能体协同是当前流行的趋势，Nexent 作为开源的智能体生成平台，同样也在平台层封装好了该能力。我们只需要接入对应的图片生成 MCP，就可以直接配置一个封面设计智能体。然后只需要在我们原来的选题智能体的基础上 `配置选择协作的智能体` 选择封面设计智能体即可，这样开源选题研究和封面设计智能体就可以合作了。

只需要在对话中输入 `帮我分析 GitHub 今天的热门开源趋势`，它就会分析今天的 GitHub 热榜，结合 HelloGitHub 的历史周刊和选题规范帮我推荐对应的选题，并自动调用封面设计智能体根据要求设计对应的封面了。

### 3.6 智能体市场

如果想要在自己的智能体集成其他人共享的智能体的功能也十分方便，只需要在智能体市场选择好自己需要的智能体，点下载就可以直接安装到自己的智能体空间，就能和自己的智能体组成一个多智能体系统啦。

当然，Nexent 的功能不止这些，还有智能体可视化关系图、多模态 SDK、多人协作等功能，而且自托管 MCP 和 智能体监测功能也在路上了，这些都是 Nexent 相较其他智能体平台更用心的地方。

四、最后
----

在实际体验 Nexent 的过程中，我有幸与 Nexent 开发团队交流，了解到了项目背后的故事。

Nexent 作为一个成立不到一年的小团队，他们思考的是更长期的问题，那就是怎么可以让 AI 普惠到更多的人。**把复杂的问题留给自己，把简单的事情交给用户**，无需一行代码，电脑小白也可以五分钟轻松构建智能体。秉持着这个理念，Nexent 一开始就选择了以 MIT 协议完全开源。老实说，这不是一条容易走的路，我也很开心看到有 Nexent 这样的开源项目仍然坚持前行着，让 AI 开源世界拥有更多中国团队的声音。

> GitHub 地址：[github.com/ModelEngine-Group/nexent](https://github.com/ModelEngine-Group/nexent)

最后，Nexent 是一个开源智能体构建平台，无需编排、复杂搭建，将自然语言转化为完整的多模态智能体应用，**一个提示词，无限种可能**。
