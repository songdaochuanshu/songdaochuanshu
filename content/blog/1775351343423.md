程序员的创业小工具 - 开源项目自荐-WeCreate
===========================

▎ 碎碎念，以及灵感来源
------------

最近一直想创业，然而没有找到需求，于是，我就成为了自己的用户，**如果诸位有合作，也可以找我**

_最近梦里都在想创业的事情，然而事实上举步维艰（主要是没有足够的money，创业失败有喝西北风的风险）_

_我估计在座的各位都是程序员，这一套应用想部署，也是分分钟的事情_

我如何辅助一位程序员（我）创业？

▎ 创业的理解（纯属个人观点）
---------------

已知创业，其实就是卖东西，无论是卖服务、卖产品还是卖其它的。

一个由资深程序员创业产生的公司，其实最重要的还是销售。只不过初期，程序员就是销售，要把自己的东西卖出去。

但是因为只有一个核心，也没有什么资金去聘请别人，尤其是创业需要投资，投资就有很高的风险亏钱。

_没有足够的资产，创业就像是送菜_

💼 假如一个最小的公司，需要的是两个角色：一个卖东西，一个生产东西（财务之类的没算上，据说在AI的帮助下，个人公司可以自行处理完成）。

如果是一人公司，尤其是卖独立网络小产品的公司。程序员作为唯一一个人去卖东西，那么就还需要有一个角色站在生产东西的定位上。

▎ WeCreate 介绍
-------------

基于这个灵感， WeCreate 开源项目就诞生了，一个利好资深程序员创业的项目

为什么是**利好**资深程序员？因为一些人直接使用AI写项目，可能会花很多时间去审核代码，对于AI产生的代码功能有一种雾里观花的感觉，很虚。

但资深程序员，对于自己熟悉的项目，简直是不要太熟悉，足够完备，避过坑的提示词，以及看AI的思考过程，如果AI带人打转，资深程序员很好看出来，然后立刻换方向。

整个项目本质就是一个**基于Project的****CLI调用管理工具，**功能说起来也很简单，

✅ 调用独立 CLI 工具，实现用户需求

✅ 基于 git 管理项目，个人派工单任务/多任务给 CLI

✅ CLI 自行实现，自行整理记忆，自行评分

⚙️ 需要在系统配置中开启记忆功能（自行整理记忆，自行评分）（会额外消耗 token），开启记忆功能后，项目的记忆文件，也可以导出或者覆盖

**基于高权限CLI的操作风险提示**

由于这个项目是相当于把电脑中的特定工作目录全交由 CLI 操作，**风险**性相对很高，但是**没有 OpenClaw 那么高**。

建议自行搭建本地 Git 仓库，且不要在公网上开放访问。

拿个不用的旧电脑，私网，或者安全受限的局域网使用即可，不要给不信任的人用，最好只自己用，需要整理需求，上传即可，其它的时间自己做其它的。

还是碎碎念一下，提示词注入很危险，所以这个输入要把控在自己手里。
