到底是我在上班，还是 OpenClaw 在上班？
========================

我现在过上了一种很离谱的生活。

飞书提醒我填工时——小龙虾自己上系统把工时填了。

老板在群里派活——小龙虾自己接完，开始主动工作了。

同事@我问个问题——小龙虾读到通知，自己去查了资料，把答案甩回去了。

有时候我都恍惚：到底是我在上班，还是小龙虾在上班？

今天这篇文章，就跟大家分享一下，我到底是怎么做到的。

* * *

先说结论。

我现在有三个 24 小时在线的 AI Agent，分别是：

*   软件小龙虾 OpenClaw：24 小时干活的主力
    
*   Claude Code：给小龙虾"修身体"的后勤保障
    
*   YoooClaw：小龙虾 OpenClaw 在手机端的代理
    

这三个东西串在一起，形成了一个自动闭环。

听起来很玄乎？

别急，一层一层给你讲。

* * *

OpenClaw — 24 小时干活的主力
=====================

小龙虾的基础知识这里就不多说了，感兴趣的可以看我之前的文章。

OpenClaw 目前就是我的主力干活工具。

需要干什么活，直接指挥它就行。

但是。

用过的朋友一定知道一个痛点：

**你让小龙虾干活没问题，但让它自己改自己的配置，它大概率会把自己搞死。**

比如让它自己给自己配一个 Agent 分身，改完配置文件再重启。

完了，重启不起来了。

因为它对自己怎么配置自己，根本就没有足够的了解。

就像你让一个外科医生给自己做手术，理论上可以，实际上大概率出事。

怎么办？

* * *

Claude Code — 小龙虾"修身体"的后勤保障
===========================

我的方案是：

用 Claude Code 打开 OpenClaw 的工作目录。

所有关于 OpenClaw 的配置变更，全部交给 Claude Code 来做。

Claude Code 改完配置后，自动重启 OpenClaw。

如果重启失败？

它自己修复，直到修完为止。

完美。

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224358357-2044003467.jpg)

上面这个就是我.openclaw目录下放的内容，其中 CLAUDE.md 就是给 Claude 每次运行时读取的系统提示词。

而 openclaw-source 则是 OpenClaw 的源码。

Claude Code 会自动进行本地源码的学习后进行配置的变更，非常稳！

但这就够了吗？

远远不够。

* * *

Happy — 随时随地远程指挥
================

新的问题来了。

我想指挥 OpenClaw 干活的时候，可以随时随地和它对话。

但是我人在外面，想变更 OpenClaw 的配置，没法用 Claude Code 怎么办？

这里就有一个顶级玩法。

**Happy**（[https://github.com/slopus/happy）是一个开源项目。](https://github.com/slopus/happy%EF%BC%89%E6%98%AF%E4%B8%80%E4%B8%AA%E5%BC%80%E6%BA%90%E9%A1%B9%E7%9B%AE%E3%80%82)

是的这个项目的名字就叫做：Happy，真的很 Happy，哈哈。

安装之后，可以直接在手机上远程操作 Claude Code。

操作页面长这个样子：

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224359173-185110170.jpg)

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224400240-1044812066.jpg)

这意味着什么？

我在地铁上、在咖啡馆、在床上，都能远程指挥 Claude Code，让它给小龙虾增加新能力、装插件、改配置。

需要干活的时候，我指挥 OpenClaw。

OpenClaw 需要编码的时候，它指挥 Claude Code。

闭环了有没有，兄弟们。

但你以为这就结束了？

重点来了。

* * *

YoooClaw — 终极自闭环
================

我日常还有一个痛点：

手机收到待办提醒后，我还得自己翻译成人话，交给小龙虾去干活。

能不能让小龙虾自己接活、自己干？

于是我给手机装了一个软件：**YoooClaw**

（目前还在内测中，预计 4 月中旬正式上线，感兴趣的可以先关注一波）

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224400894-260004149.jpg)

这个软件绝了。

它可以读取我手机上的所有未读通知，然后把通知转发给云端的 OpenClaw。

OpenClaw 接到通知后，自己判断该干嘛，然后直接干。

这个场景非常 Nice，我举个例子：

我每天手机里面的未读通知有很多，单单今天上午的未读通知就有 31 个。

只要点击一键喂虾，YoooClaw 就会自动将手机的未读通知全部进行提取和总结。

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224401551-143487720.jpg)

![图片](https://img2024.cnblogs.com/blog/1104472/202604/1104472-20260407224402221-993955183.jpg)

对于一些信用卡还款，快递待取件这些场景，可真的太舒服了。

不然我还要一个一个翻未读通知来确定今天还有哪些事情没做，真的很烦。

除此之外，因为 YoooClaw 本身可以获取我的所有未读消息。

所以我可以针对关键通知进行配置。

当飞书提醒我 OA 工时未填写，此时 YoooClaw 读到这条通知，会自动转发给云端的 OpenClaw。

OpenClaw 则自己登录工时系统，把工时给填了。

全程我一个字没打，哈哈。

YoooClaw 作为我的事件触发端来 push 小龙虾来干活，真的爽歪歪。

类似于 OpenClaw 本身是在云端，他没办法接触你最重要的手机设备信息。

但是 YoooClaw 则弥补了这一环，让我的 OpenClaw 直接和手机端消息进行了打通。

除此之外，YoooClaw 由于是手机端的设备，所以还支持使用硬件小卡片，直接来进行录音总结，并丢给小龙虾去执行后续的工作流。

可玩性非常多，等我后续探索完，到时候给大家细细道来。

* * *

最后说两句
=====

很多人问我，小龙虾到底能干嘛？

我觉得这个问题本身就问反了。

不是小龙虾能干嘛，是你愿意让它帮你干嘛。

3 月份小龙虾热度最高的时候，全网都在围观，都在讨论"它会不会替代我"。

现在热度过了，围观的人散了。

但留下来的人，开始发现它真正的价值。

不是那些花里胡哨的演示视频，而是像今天这样。

安安静静地帮你把工时填了，把日报写了，把活儿干了。

**这才是 AI 该有的样子，不是替代你，而是安静地站在你身后帮你干事。**

至于今天分享的这套体系，有不清楚的可以评论区告诉我，我们评论区见！

越是在这种平淡期，越是普通人沉淀自己的好时候。

**看了很多内容，但仍然还没用过小龙虾的朋友，不妨亲自去试一试。**

**毕竟现在各个大厂都推出了不同形态的小龙虾，大家的可选面多了很多。**

今年必然是Agent 元年，没得跑了，大家一起加油呀！
