可对话的赛博分身：用 Claude Code 分析 GitHub 日记
===================================

前言
--

在这个数字时代，我希望拥有一个能够记录、分析我的行为模式的智能助手。经过尝试，我构建了一个免费开放的日记系统——它不仅能记录我的日常，还能通过 AI 进行深度分析，成为我的"赛博分身"。

我的痛点和需求
-------

### 传统日记应用的局限性

1.  **付费壁垒**：很多功能完善的日记应用，高级功能需要额外付费
2.  **格式封闭**：使用私有格式存储数据，导致难以迁移和导出
3.  **缺乏智能分析**：无法与 AI 集成，无法进行深度查询和分析
4.  **同步限制**：不同设备间的同步往往需要额外费用

### 我的理想方案

我需要一个满足以下条件的解决方案：

*   ✅ **完全免费**：包括同步和所有高级功能
*   ✅ **开放标准**：使用通用格式，确保数据的永久性和可迁移性
*   ✅ **AI 友好**：可以轻松集成 LLM 进行智能分析和对话
*   ✅ **多端支持**：手机、电脑都能方便地编辑和查看

解决方案架构
------

### 技术栈选择

    GitHub 仓库（核心存储）
    ├── WebStorm（电脑端编辑）
    ├── PuppyGit（安卓端 Git 操作）
    ├── Markor（安卓端 Markdown 编辑）
    └── Claude Code + GLM-4.6（AI 分析引擎）
    

### 为什么是这个组合？

**GitHub 作为存储后端**

*   免费的私有仓库
*   版本控制和备份

**Markdown 作为存储格式**

*   纯文本，永不过时
*   支持简单样式
*   所有平台都有优秀的编辑器
*   AI 友好，易于解析和分析

**Claude Code + GLM-4.6 作为分析引擎**

*   Claude Code 提供强大的上下文能力
*   GLM-4.6 拥有较大的上下文窗口（适合长文本分析）
*   性价比高，适合日常频繁使用

### 其他尝试

在构建这个系统的过程中，我也尝试了其他方案：

**GitJournal**

*   集git操作和markdown编辑器为一体的安卓app
*   但bug较多，且很久没有维护更新了，所以放弃了。替换成了 PuppyGit + Markor 的方案。

实施细节
----

### 目录结构设计

    diary/
    └── 2025-01/                    # 按年-月组织的日记目录
        ├── 2025-01-01-08.md        # 日记文件，使用时间命名
        ├── 2025-01-02-09.md
        └── ...
    └── 2025-02/
        └── ...
    

### Front-matter 规范

每个日记文件都包含结构化的元数据：

    ---
    tags: [饮食, 思考, 人生感悟]
    ---
    
    这里是日记正文...
    

**优点：**

*   便于筛选和分类
*   支持 AI 进行主题分析
*   保持文件整洁

### 多端工作流

**电脑端（主要编辑）**

1.  WebStorm 打开项目
2.  使用 LLM 生成 tags
3.  基于日记内容分析和对话
4.  推送到 GitHub

**手机端（快速记录）**

1.  Markor 创建/编辑日记
2.  PuppyGit 执行 git 操作
3.  推送到 GitHub

AI 分析能力展示
---------

这是整个系统最有趣的部分——让你的日记"活"起来。

### 时间线查询

**示例提问：**

> "我今年主要在关注什么话题？"

AI 会扫描所有相关日记，提取关键话题，生成趋势分析：

    ⏺ 根据对你 2025 年日记的分析，我发现你主要关注以下几个话题：
    
    主要关注领域
    
    1. 思考与反思（最频繁）
    - 对生活、工作、人际关系的深入思考
    - 社会现象的观察与反思
    - 个人成长与自我认知
    
    2. 日常生活记录
    - 饮食与健康密切相关
    - 摩托车的使用与保养
    - 通勤与交通问题
    
    3. 精神追求
    - 诗歌创作与欣赏
    - 哲学思辨
    - 人生感悟与自我和解
    
    4. 技术探索
    - 日记软件的研究与选择
    - 工具的使用与优化
    - 科技产品的观察与思考
    
    5. 人际关系
    - 家庭关系（尤其是与父母的相处）
    - 育儿经验的记录
    - 朋友聚会与社交活动
    
    话题特点
    
    你的日记体现了很强的内省性，经常记录深度的思考和感悟。同时也很注重生活细节，特别是饮食和健康。你对技术和工具保持持续的兴趣，并且有精神层面的追求，比如诗歌和哲学。
    
    

### 模式识别

**示例提问：**

> "我的饮食习惯有什么变化规律？"

    ⏺ 基于对日记内容的分析，我发现了你的以下几个饮食习惯变化规律：
    
    饮食习惯分析报告
    
    1. 社交饮食模式
    
    - 与朋友聚会时容易饮食过量（如...）
    - 但会有意识地通过走路等方式来弥补（吃完走了15分钟路）
    - 更愿意尝试新的餐厅和美食（如纸包鱼、光谷东的餐厅）
    
    2. 家庭饮食影响
    
    - 对家人（特别是...）的买菜习惯有不满（...）
    - 注重食材的消化问题（发现...）
    
    3. 个人饮食习惯总结
    
    - 容量控制意识：知道自己的食量极限（...）
    - 健康关注：会记录食物对身体的影响（...等）
    - 饮食调节小技巧：发现了一些缓解不适的方法（...）
    
    4. 时间规律
    
    - 从记录看，你从2024年9月开始有意识寻找美食
    - 饮食相关记录在10月和12月较为集中
    
    5. 情绪与饮食的关系
    
    虽然记录有限，但可以看出：
    - 社交场合容易饮食过量
    - 事后会有后悔情绪，但会采取补救措施
    - 对健康饮食有一定认知，但执行时会受社交影响
    
    建议：
    1. 社交聚会前可以先吃些健康食物，避免过量
    2. 继续保持饮食后的补救习惯（如散步）
    3. 可以考虑记录更多日常饮食，以便更好地分析模式
    

### 提示词设计最佳实践

为了获得更好的分析结果，我总结了一些提示词技巧：

**1\. 结构化查询**

    请分析我最近的日记，关注以下方面：
    1. 学习新技能的情况
    2. 健康相关记录
    3. 情绪变化趋势
    请用表格形式总结，并指出需要注意的模式（如拖延症、情绪波动、不良习惯等）
    

**2\. 情感分析提示**

    分析这段时期的情绪变化趋势：
    - 标注正面/负面情绪的关键事件
    - 识别情绪触发因素
    - 提供3-5个具体的情绪管理建议
    

**3\. 目标追踪**

    检查我在[具体目标]上的进展：
    - 记录的相关活动频率
    - 遇到的困难和突破
    - 下一步行动建议
