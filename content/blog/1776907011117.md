Kimi 新模型发布！教你如何在 Claude Code 上配置并使用最新的 k2.6 模型！
===============================================

前言
==

Kimi k2.6 是月之暗面推出的新一代代码大模型，在代码生成、Agent 能力和长上下文理解上都有显著提升。虽然 Kimi 订阅计划支持在 Claude Code 上进行配置使用，但此前 Kimi k2.5 在 Claude Code 中接入的配置方式已失效。无论你是想寻找 Claude 的平价替代，还是 Kimi + Claude Code 的使用者，都可以通过本篇文章了解到如何在 Claude Code 上配置与使用 Kimi k2.6 模型。

本文将系统性地介绍从零开始的完整配置流程，以及实际部署过程中遇到的典型问题及其解决方案。

**适用版本**：Claude Code ≥ 2.1.x，Kimi k2.6（模型 ID：`kimi-for-coding`）

* * *

一、准备工作
======

1.1 安装 Claude Code
------------------

如果你已经安装过 Claude Code，可以跳过这一步。

**macOS / Linux：**

    # 安装 Node.js（推荐 24.x）
    curl -fsSL https://fnm.vercel.app/install | bash
    # 重新打开终端后执行
    fnm install 24.3.0
    fnm default 24.3.0
    fnm use 24.3.0
    
    # 安装 Claude Code
    npm install -g @anthropic-ai/claude-code
    
    # 初始化配置（跳过首次引导）
    node --eval "
    const os = require('os'), fs = require('fs'), path = require('path');
    const fp = path.join(os.homedir(), '.claude.json');
    const c = fs.existsSync(fp) ? JSON.parse(fs.readFileSync(fp)) : {};
    fs.writeFileSync(fp, JSON.stringify({ ...c, hasCompletedOnboarding: true }, null, 2));
    "
    

**Windows（PowerShell）：**

    winget install --exact --source winget --id Git.Git
    winget install --exact --source winget --id OpenJS.NodeJS
    # 重新打开终端
    npm install -g @anthropic-ai/claude-code
    

验证安装：

    claude --version
    # 应输出类似：2.1.117 (Claude Code)
    

1.2 获取 Kimi Code API Key
------------------------

前往 Kimi 官网购买会员订阅计划：[https://www.kimi.com/](https://www.kimi.com/)

访问 Kimi 会员控制台创建并获取 API Key：[https://www.kimi.com/code/console](https://www.kimi.com/code/console)

点击「创建 API Key」，复制以 `sk-kimi-` 开头的密钥。这个 Key 是后续连接 Kimi k2.6 的凭证，请妥善保管。

![image](https://img2024.cnblogs.com/blog/1483609/202604/1483609-20260422191142531-1202246961.png)

> **注意**：Kimi Code 平台（`api.kimi.com`）和 Moonshot 开放平台（`api.moonshot.cn`）属于独立的认证体系。在该页面获取的 API Key 仅适用于 `api.kimi.com/coding/` 端点，不可用于 `api.moonshot.cn`。

二、核心配置
======

Claude Code 通过读取环境变量来决定连接哪个模型后端。根据 Kimi 官方最新文档，正确的配置方式如下：

2.1 写入 Shell 配置文件
-----------------

根据你的终端类型，编辑对应的配置文件：

*   **Zsh**（macOS 默认）：`~/.zshrc`
*   **Bash**：`~/.bashrc` 或 `~/.bash_profile`

在文件末尾添加以下内容：

    # ============================================
    # Claude Code + Kimi k2.6 配置
    # ============================================
    export ANTHROPIC_BASE_URL="https://api.kimi.com/coding/"
    export ANTHROPIC_API_KEY="sk-kimi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"  # 替换为你的 Key
    export ANTHROPIC_MODEL="kimi-for-coding"
    export ANTHROPIC_DEFAULT_OPUS_MODEL="kimi-for-coding"
    export ANTHROPIC_DEFAULT_SONNET_MODEL="kimi-for-coding"
    export ANTHROPIC_DEFAULT_HAIKU_MODEL="kimi-for-coding"
    export CLAUDE_CODE_SUBAGENT_MODEL="kimi-for-coding"
    export ENABLE_TOOL_SEARCH="false"
    

使配置生效：

    source ~/.zshrc
    

2.2 写入 Claude Code 用户配置（可选但推荐）
------------------------------

Claude Code 也会读取 `~/.claude/settings.json` 中的 `env` 字段。写入后，即使在不加载 Shell 环境的情况下（如从 GUI 启动），也能正确识别配置。

    # 确保目录存在
    mkdir -p ~/.claude
    
    # 写入配置
    cat > ~/.claude/settings.json << 'EOF'
    {
      "skipDangerousModePermissionPrompt": true,
      "env": {
        "ANTHROPIC_BASE_URL": "https://api.kimi.com/coding/",
        "ANTHROPIC_API_KEY": "sk-kimi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
        "ANTHROPIC_MODEL": "kimi-for-coding",
        "ANTHROPIC_DEFAULT_OPUS_MODEL": "kimi-for-coding",
        "ANTHROPIC_DEFAULT_SONNET_MODEL": "kimi-for-coding",
        "ANTHROPIC_DEFAULT_HAIKU_MODEL": "kimi-for-coding",
        "CLAUDE_CODE_SUBAGENT_MODEL": "kimi-for-coding",
        "ENABLE_TOOL_SEARCH": "false"
      }
    }
    EOF
    

> **安全提示**：建议将 `.claude/settings.json` 的权限设置为 `600`，以防止其他系统用户读取 API Key。

三、验证配置
======

3.1 检查环境变量
----------

    env | grep ANTHROPIC
    

应输出类似：

    ANTHROPIC_BASE_URL=https://api.kimi.com/coding/
    ANTHROPIC_API_KEY=sk-kimi-...
    ANTHROPIC_MODEL=kimi-for-coding
    

3.2 启动 Claude Code
------------------

进入任意项目目录，启动 Claude Code：

    cd ~/your-project
    claude
    

在 Claude Code 的交互界面中输入：

    /status
    

如果看到类似以下输出，说明配置成功：

四、故障诊断与配置问题
===========

4.1 端点配置错误
----------

部分历史文档推荐使用 `https://api.moonshot.cn/anthropic` 作为 `ANTHROPIC_BASE_URL`，并配合 `ANTHROPIC_AUTH_TOKEN` 进行认证。该配置在当前环境下已不适用。

**症状**：API 请求返回 `401 Invalid Authentication`，即使用户确认 API Key 在平台控制台中处于有效状态。

**根因**：`api.moonshot.cn` 属于 Moonshot 开放平台的独立认证体系，而 `sk-kimi-` 前缀的 API Key 由 Kimi Code 平台签发，仅对 `api.kimi.com` 域下的端点有效。两个平台之间的凭证不互通。

**解决方案**：

    export ANTHROPIC_BASE_URL="https://api.kimi.com/coding/"
    export ANTHROPIC_API_KEY="sk-kimi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
    export ANTHROPIC_MODEL="kimi-for-coding"
    

4.2 模型标识符错误
-----------

Kimi k2.6 在 API 层面的内部模型标识符为 `kimi-for-coding`，而非 `kimi-k2.6`。Claude Code 在不显式设置 `ANTHROPIC_MODEL` 时，默认会请求 `claude-sonnet-4-6` 等 Anthropic 原生模型名。Kimi Code 后端会对这些 Claude 模型名执行自动映射，将其路由至 `kimi-for-coding`。因此，不设置 `ANTHROPIC_MODEL` 亦可正常工作；若显式配置，则必须使用 `kimi-for-coding`。

**症状**：请求返回 `404 model_not_found` 或类似的资源不存在错误。

**验证方法**：

    curl -s https://api.kimi.com/coding/v1/models \
      -H "Authorization: Bearer $ANTHROPIC_API_KEY" | jq '.data[].id'
    

预期输出仅包含 `kimi-for-coding`。

4.3 配置生效但请求无响应
--------------

`/status` 显示 `Anthropic base URL` 与 `Model` 均正确，但提交消息后长时间无响应或连接超时。

**排查清单**：

1.  **网络连通性**：确认终端可以正常解析并访问 `api.kimi.com`。执行以下命令验证：
    
        curl -I https://api.kimi.com/coding/v1/models
        
    
    应返回 HTTP 200。
    
2.  **账户余额与配额**：登录 [https://platform.kimi.com/console/api-keys](https://platform.kimi.com/console/api-keys) 确认账户存在有效余额或可用额度。额度耗尽时服务端通常返回 `429 Too Many Requests` 或 `403 Forbidden`，但在某些边界情况下也可能表现为连接挂起。
    
3.  **Claude Code 版本兼容性**：Claude Code 2.1.x 以下版本对第三方 base URL 的支持存在已知问题。建议升级至最新版本：
    
        npm install -g @anthropic-ai/claude-code

五、总结
----

将 Claude Code 切换到 Kimi k2.6 的核心就三步：

1.  **获取 Key**：在 [https://www.kimi.com/code/console](https://www.kimi.com/code/console) 创建 `sk-kimi-` 开头的 API Key
2.  **改配置**：将 `ANTHROPIC_BASE_URL` 指向 `https://api.kimi.com/coding/`，使用 `ANTHROPIC_API_KEY` 认证
3.  **验连接**：启动 `claude`，输入 `/status` 确认模型为 `kimi-for-coding`

配置完成后，你就能在熟悉的 Claude Code 界面中，享受 Kimi k2.6 带来的代码生成与 Agent 能力了。
