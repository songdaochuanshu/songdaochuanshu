腾讯小龙虾WorkBuddy技能与插件深度解析
=======================

腾讯小龙虾（WorkBuddy）里的「技能」和「插件」到底是干嘛的？
==================================

> 作者背景说明：本文面向有一定编程基础（比如懂 Python 和 Arduino/STM32）但不是专职软件开发者的用户，尽量用类比的方式把原理讲清楚。

* * *

用过腾讯小龙虾（WorkBuddy / CodeBuddy）的朋友，大概都知道它有三个核心模块：**自动化（Automations）**、**技能（Skills）**、**插件（Plugins）**。

自动化好理解——定时触发任务，跟 Arduino 的定时中断一个道理。但「技能」和「插件」很多人搞不清楚，感觉都是在"增强 AI 能力"，到底有什么区别？怎么用？

这篇文章就把这两个模块说透。

* * *

先说自动化——基准线
----------

类比 Arduino 的定时任务：**每隔 X 时间，自动触发一件事**。

比如"每天早上 9 点，帮我检查一下项目进度"。自动化不增加新能力，只是把你平时手动触发的操作变成定时自动执行。它是一个**调度器**，而不是能力扩展器。

理解了这个，技能和插件的定位就更容易区分了。

* * *

技能（Skills）是什么？
--------------

### 一句话定义

**技能 = 给 AI 注入领域专业知识的「说明书」**

### 类比：Python 的 import

你在 Python 里写 `import pandas`，之后你就"会用 pandas"了——你获得了处理表格数据的能力。

技能干的是同一件事，只不过给的不是代码库，而是**一份写给 AI 的专业说明书**（一个叫 `SKILL.md` 的文件）。

这份说明书里包含：

*   这个领域的背景知识
*   处理这类任务的标准流程（SOP）
*   常见坑和注意事项
*   推荐使用的工具和方法

### 具体例子

默认的 AI 助手不太懂怎么正确创建 PowerPoint——它可能用错 API、格式混乱、图片处理出问题。但安装了「pptx 技能」之后，AI 就加载了一份详细说明书，里面写着用什么库、怎么处理图片、怎么设置排版……它立刻变成了一个"PPT 专家"。

类似的技能还有：

*   **xlsx 技能**：让 AI 更擅长操作 Excel 表格
*   **pdf 技能**：让 AI 更擅长处理 PDF 文件
*   **小红书内容技能**：让 AI 按小红书风格写文案
*   **pptx 技能**：让 AI 正确生成演示文稿

### 技能的本质

> 技能不增加新工具，但让 AI 在特定领域**更聪明、更专业、更不容易犯错**。

这是知识层面的增强。

### 触发方式

AI 会根据你的请求自动判断要不要加载某个技能，或者你直接说"用 xxx 技能帮我……"来明确指定。

* * *

插件（Plugins / MCP）是什么？
---------------------

### 一句话定义

**插件 = 给 AI 接上真实世界操作接口的「功能模块」**

### 类比：给 Arduino 接外部模块

Arduino 本身只能做有限的事。但你插上一个 GPS 模块，它就能获取位置；插上一个 Wi-Fi 模块，它就能联网；插上一个蓝牙模块，它就能和手机通信。

**插件给 AI 接上了原本不存在的能力接口**——让 AI 能访问外部数据、调用外部工具、执行真实操作。

### MCP 协议是什么？

这里有个技术名词叫 **MCP（Model Context Protocol，模型上下文协议）**。

你可以把它理解成一个**标准接口协议**——就像 Arduino 的引脚定义标准，让各种模块都能按同一套规范插上去，不用每次重新发明接口格式。

有了 MCP 这个标准，任何人都可以开发插件，AI 助手也能用统一的方式调用各种插件，生态就这么建立起来了。

### 插件能做什么

插件类型

类比 Arduino 模块

实际效果

微信公众号搜索插件

Wi-Fi 模块

AI 能主动去公众号搜文章

腾讯云 API 插件

云服务模块

AI 能直接操作你的云服务器

### 插件的本质

> 插件让 AI 从一个「只会说话的助手」变成「能动手操作的执行者」。

这是能力层面的扩展。

### 触发方式

插件装上后在后台待命，无需手动触发。你直接提出需求，AI 判断需要调用哪个插件就会自动用。**关键是要先装好插件**，没装的能力就是真的没有。

三者关系，一张图说清楚
-----------

    你的请求
        │
        ▼
    ┌─────────────────────────────────────────┐
    │           AI 助手（小龙虾）              │
    │                                         │
    │  + 技能（Skills）                        │  ← 让 AI 更聪明、更专业
    │    = 领域知识 + 操作规范注入              │    （知识层面）
    │                                         │
    │  + 插件（Plugins / MCP）                 │  ← 给 AI 新的能力接口
    │    = 调用外部工具 / 执行真实操作          │    （能力层面）
    │                                         │
    └─────────────────────────────────────────┘
        │
        ▼
    自动化（Automations）= 上面这一切的定时触发器
    

**一句话总结三者区别：**

*   **自动化**：_什么时候_做
*   **技能**：_更擅长怎么_做（知识层面）
*   **插件**：_能做什么_（能力层面）

* * *

怎么用好？实用建议
---------

### 技能的使用建议

**遇到专业领域任务，先看有没有对应技能。**

处理 Excel、生成 PPT、写小红书文案、操作 PDF——这些都有专门技能。直接说"用 xxx 技能帮我……"，效果会比不指定技能强很多。

技能就像你请了个专科医生，比全科医生在特定领域靠谱得多。

### 插件的使用建议

**插件装上就在后台待命，无需手动触发。**

你只需直接提出需求，比如"帮我搜一下微信公众号上关于 MCP 的文章"——如果装了对应插件，AI 自然就会调用。

重点是**提前把常用插件装好**，这是一次性的配置投入，之后就能持续享受能力增强。

### 组合使用才是最强姿势

技能 + 插件 + 自动化可以叠加，形成完整工作流：

> 「每周一（**自动化**）用微信搜索插件（**插件**）搜集行业动态，用小红书内容技能（**技能**）整理成一份推文草稿」

这就是三个模块协同工作的典型场景。
