### 一、这不是简单的“套壳”

Solon Code CLI 不是简单模仿 Claude Code CLI 的表面功能，而是从基础（Solon AI）到内核再到交互，全方位对齐 Claude Code 的设计理念，并针对 Java 开发场景做了深度优化。采用 MIT 开源协议，可以任意修改与定制。

### 二、极简交互，极致体验

参考 README.md 修改 llm 配置后（可以给它取个昵称：“小花”），运行 `run.sh`（mac or linux） 或 `run.cmd`（window），你就进入了一个全新的世界：

    小花 v0.0.12
    /Users/yourname/projects/myapp/
    
    
    User
    > 
    

没有繁杂的命令参数，没有复杂的配置。就像和一个经验丰富的程序员对话，你只需要说出你的需求。

#### 3\. 安全的命令执行

对于高危操作，Solon Code CLI 内置了 Claude Code 风格的安全审计策略：

    User
    > 获取 https://solon.noear.org/ 的网页内容
    
    小花
      用户想要获取 https://solon.noear.org/ 这个网页的内容。我需要使用一个能够获取网页内容的工具。
      
      看一下可用的工具，我看到有这些相关的工具：
      1. `bash` - 可以执行 shell 命令，比如 curl 或 wget
      2. `full_text_search` - 这是本地全文搜索，不是获取网页内容的
      3. 其他工具主要是文件操作相关的
      
      最直接的方法是使用 `bash` 工具执行 curl 命令来获取网页内容。
      
      
      我来帮你获取这个网页的内容。
       (8554 tokens)
    
    Permission Required
    Command: curl -s https://solon.noear.org/
    Approve? (y/n)
    

它（通过可定制的拦截策略 HitlStrategy）会智能拦截：

    系统特权命令（sudo、chmod、chown）
    
    网络外连操作（curl、wget、ssh）
    
    路径回溯攻击（../）
    
    组合命令注入（;、&）
    
    等...
    

只有经过你确认的高危操作才会执行，安全性与 Claude Code 完全对齐。

### 五、三大连接模式，无处不在的 AI

Solon Code CLI 采用了“池盒模型”（Pool-Box Model），一个 AI 核心，三种连接方式：

直接在命令行中与 AI 对话，最纯粹的体验。

#### 1\. Cli 模式（通过配置启用）

    solon.code.cli:
      cliEnabled: true
      cliPrintSimplified: true # 是否简化打印
    

Cli 模式，默认配置为 true （即启用）

#### 2\. Web 模式（通过配置启用）

    solon.code.cli:
      webEnabled: false
      webEndpoint: "/cli"
    

通过浏览器访问 `http://localhost:8080/cli`，获得可视化的交互界面。

#### 3\. ACP 协议模式（通过配置启用）

Solon Code CLI 实现了 ACP（Agent Communication Protocol）协议，可以无缝对接任何支持该协议的 IDE 插件或客户端。这意味着：

未来可以在 VS Code、IntelliJ IDEA 中直接使用

支持远程连接，AI 在你的服务器上运行，你在本地操作

团队共享 AI 计算资源

    solon.code.cli:
      acpEnabled: false
      acpTransport: "websocket" # "stdio" or "websocket"
      acpEndpoint: "/acp"
    

#### 技术栈透明，随你所用（MIT 协议，还可以随意定制）

Solon Code CLI 不绑定特定技术栈：

    // Maven 项目自动识别
    if (exists("pom.xml")) {
        addCommand("Build: `mvn clean compile`");
        addCommand("Test: `mvn test`");
    }
    
    // Node 项目自动识别  
    if (exists("package.json")) {
        addCommand("Install: `npm install`");
        addCommand("Test: `npm test`");
    }
    
    // 子模块智能处理
    if (Files.exists(dir.resolve("pom.xml"))) {
        addModule(dirName + " (Maven module)");
    }
    

无论你是 Java 后端、Node.js 前端，还是 Go 微服务等等，它都能理解你的项目。

### 六、快速开始

下载地址（17MB）：

    https://github.com/opensolon/soloncodecli/releases/download/v0.0.12/SolonCodeCLI_v0.0.12.zip
    https://gitee.com/opensolon/soloncodecli/releases/download/v0.0.12/SolonCodeCLI_v0.0.12.zip
    

（下载后解压，参考 README.md 调整配置）主要配置：

    solon:
      code:
        cli:
          nickname: "小花"
          workDir: "my-project"
          chatModel:
            apiUrl: "https://api.deepseek.com/v1/chat/completions"
            apiKey: "sk-xxxx"
            model: "deepseek-chat"
          planningMode: true
          hitlEnabled: true
    

运行

    # sh 命令（for mac or linux）
    sh run.sh
    
    # cmd 命令（for window）
    run.cmd
    
    # （或者）原生 java 命令
    java -jar SolonCodeCLI.jar
