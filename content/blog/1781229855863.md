二、界面介绍
------

### 1、整体布局

打开 Reasonix 桌面端，界面分三栏：![image.png](https://img2024.cnblogs.com/blog/3418714/202606/3418714-20260611161717446-1365756338.png)

*   左侧：会话列表，支持在应用内新建、切换、删除会话
*   中间：主聊天区，Agent 的推理过程、工具调用、文件修改全部在这里流式呈现
*   右侧：文件面板，实时记录当前会话中 Agent 读取或修改过的所有文件
*   底部：状态栏，常驻显示成本仪表盘这个布局设计有明确的意图让你能随时知道它在做什么、花了多少钱。

### 2、多标签会话

桌面端支持同时打开多个标签，每个标签对应一个独立的 Agent 会话，可以并行处理不同项目。标签之间完全隔离，后台标签的键盘快捷键不会被误触发到前台。重启应用后，所有标签、会话状态、滚动位置自动恢复，不用重新打开项目。

### 3、右侧文件面板

这个面板解决了一个使用 Agent 时的常见焦虑：不知道 Agent 动了哪些文件。  
面板实时列出当前会话中所有被 Agent 读取或编辑的文件，点击可以直接查看内容。Agent 调用 remember / forget 工具后，记忆面板也会在同一位置实时更新，不需要等到下一轮对话刷新。  
输入框支持 @ 提及文件，可以拖拽文件进去，也可以用树形选择器浏览项目目录。

### 4、底部成本仪表盘

这是整个桌面端最有特色的设计，也是 Reasonix 与竞品差异最大的地方。  
状态栏实时显示三个指标：

*   缓存命中率：当前会话的前缀缓存命中比例
*   本轮 Token 消耗：当前这一轮请求的 Token 用量
*   费用：按轮和按会话分别统计，颜色编码直观，绿色低于 $0.05，黄色 $0.05 到 $0.20，红色超过 $0.20  
    每一笔费用都是可见的，如果某轮任务触发了从 flash 到 pro 的自动升级，头部会出现醒目的黄色提示"⇧ pro escalated"，告诉你这轮为什么贵。

### 5、Plan Mode 与审批流程

在执行文件修改、Shell 命令等破坏性操作之前，Agent 会提交执行计划，等待用户确认。  
桌面端以弹窗 Modal 形式呈现这个确认环节，比在终端里看一大段文字清晰得多。可以逐步批准，也可以直接拒绝并给出修改方向，Agent 会在原计划基础上调整后重新提交。

### 6、斜杠命令

输入框输入 / 会弹出命令补全菜单，常用命令：

命令

作用

/pro

下一轮临时升级到 v4-pro，用完自动复位

/compact

手动压缩当前上下文

/retry

重试上一轮

/btw

追加背景信息，不触发新一轮推理

/preset

切换 fast / smart / max 预设档位

/skill

调用内置或自定义 Skill 脚本

/feedback

反馈入口

> /pro 值得单独说一下：它的设计是"用完自动解除"。你预判某个任务需要更强的推理，输入 /pro，头部出现黄色"⇧ pro armed"标识，下一轮执行完毕后自动回到默认档，不需要手动切回去。这样避免了"忘记切回 flash"导致账单异常的问题。

### 7、设置与个性化

![image.png](https://img2024.cnblogs.com/blog/3418714/202606/3418714-20260611161714824-895200364.png)  
设置面板在应用内直接打开，几个值得关注的选项：

*   API Key 和 Base URL 始终可见，不藏在折叠区
*   字体大小和字体家族可调，长时间使用的开发者会需要这个
*   深色 / 浅色主题切换
*   语言切换，含简体中文，设置后界面和工具描述全部切换

### 8、自动更新

桌面端内置更新检查，About 弹窗一键触发检测，发现新版本后可以直接在应用内下载安装，下载进度条可见。目前桌面端版本迭代很快，基本每隔一两天有新 Release，自动更新是个高频功能。
