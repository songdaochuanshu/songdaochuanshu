Hermes官方桌面版发布了
==============

大家对 `Hermes Agent` 应该不陌生了。

`Nous Research` 开源的 AI 智能体项目，MIT 协议，GitHub 上 star 已经突破了 18 万。OpenRouter 应用排行榜上也是断层第一。

GitHub：[https://github.com/NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent)  
官网：[https://hermes-agent.nousresearch.com/](https://hermes-agent.nousresearch.com/)

![hermes官方地址](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733009-772750031.png)

核心卖点就三个词：**持久记忆 + 自我进化 + 全平台消息接入**。支持 Telegram、Discord、Slack、WhatsApp、Signal，以及国内的飞书、钉钉、微信、QQ——你配置好之后，它就像一个真正的助手，在所有消息平台上等你的指令。

官方定义非常干脆：**The agent that grows with you** —— **一个会随着使用不断成长的 Agent**。

![Hermes应用排行榜](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733113-867803818.png)

* * *

跟之前养「小龙虾」（OpenClaw）一样，部署 Hermes Agent 让它帮我们干活，社区里叫「养马」——Hermes 既是希腊神话里的赫尔墨斯，又与爱马仕同名，你打开终端跑 Hermes 就像在喂马，用得越多它就越聪明。

但以前，「养马」只有一个入口：命令行。

安装要敲命令，配置网关要敲命令，选模型要敲命令，启动还得敲命令。光是下面这一屏，就劝退了无数想试试的小白：

    hermes              # 交互式命令行界面 — 开启对话
    hermes model        # 选择大语言模型服务商与对应模型
    hermes tools        # 配置启用的工具集
    hermes config set   # 设置单项配置项
    hermes gateway      # 启动消息网关
    hermes setup        # 运行全量配置向导
    hermes claw migrate # 从OpenClaw迁移数据
    hermes update       # 更新至最新版本
    hermes doctor       # 诊断运行环境与配置问题
    

非程序员看到终端两个字，基本就关窗口了。

![安装Hermes](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733022-468534200.png)

一个社区开发者，改变了局面
-------------

今年 4 月，一个叫 **fathah** 的社区开发者做了一个东西：**Hermes Desktop**。

他把 Hermes Agent 和 Web UI 打包进一个 Electron 外壳，做成了双击就能启动的原生桌面应用。

GitHub：[https://github.com/fathah/hermes-desktop](https://github.com/fathah/hermes-desktop)

![GitHub地址](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733084-588380626.png)

Mac 用 .dmg，Windows 用 .exe，Linux 有安装脚本。

![各种系统安装](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733073-1455865953.png)

一个「就是把终端应用包了个桌面壳」的项目，**20 多万人下载**。

这个数据一出来，你就知道需求有多真实了——不是「我们可能需要一个桌面版」，是「我们太需要一个桌面版了」。

Nous Research 做了一个聪明的决定
-----------------------

不是收购，不是合作，不是 fork 一个官方版跟社区打擂台。

**而是把 fathah 的整个 Hermes Desktop 项目，直接纳入官方产品线。**

上线了专属下载页面，三大主流系统全部支持。

![官方下载](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733124-2132089347.png)

![三大主流系统都支持](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733165-1308823899.png)

> 贴心的我也替你们准备了网盘备份：  
> 链接：[https://pan.quark.cn/s/fd14ee9eb927](https://pan.quark.cn/s/fd14ee9eb927)  
> 提取码：4P89

并且把 Hermes Desktop 定位为跟 CLI 和消息平台并列的**第三大使用场景**。

这个决定本身，比 Hermes Desktop 更有意思。

我们看过太多社区项目跟官方博弈的故事了——Red Hat 和 CentOS、Oracle 和 Hudson……很多时候是社区做出来一个好东西，官方觉得被威胁了，要么冷处理，要么自己做一套把社区耗死。但 Nous Research 选了另一条路：

**你做得对，你来，我帮你。**

这种开放心态，在 AI 圈子里太难得了。

真实安装体验
------

下面是我装完之后的实际感受，没有美颜。

安装过程对首次接触 Hermes 的同学来说可能会有点慢——它会把运行需要的各种环境一起打包安装。耐心等就行。

![安装页面](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733017-1231378892.png)

装完之后，实际用起来大致是这个感觉：

![Hermes Desktop 操作演示](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075734882-335995296.gif)

打开之后的第一感觉——这也太像 Codex 了吧。

左侧菜单栏，右侧对话区，整体风格跟 Codex 桌面版几乎一个模子。但我觉得换个角度看这是好事：就像换新手机，不需要重新学怎么用，上手就是熟悉的感觉。

![安装完成后](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260615075733064-1643563677.png)

如果之前装过 Hermes，桌面端会自动导入你的历史 session，不用从零开始。

### ⚠️ 踩坑预警

装是装上了，但不是一帆风顺。我踩了几个坑，列出来供参考：

1.  **首次启动可能卡在加载界面** —— 如果等了超过 5 分钟还没进去，关掉重开一般能解决。本质上是它后台在拉依赖，偶发性卡住。
2.  **Windows 下杀毒软件可能误报** —— Electron 打包的 exe 有概率触发 Windows Defender，添加信任即可。
3.  **旧版 CLI 配置文件的兼容问题** —— 如果你之前用 CLI 配过比较复杂的自定义配置，桌面版初次启动时可能读取失败。建议先备份 `~/.hermes/config.yaml`，让桌面版重新生成一份干净的再手动迁移。

坑不算大，但提前知道能省不少时间。

「把终端换成了图标」
----------

过去这一年，Claude Code、Codex CLI、OpenAI Codex……这些 Agent 产品一个比一个强，但都有一个共同点：**你得开终端**。

如果你不是程序员的人，看到终端两个字就已经放弃了。

而 Hermes Desktop 做了一件非常简单的事：

**把终端换成了图标。**

双击，打开，打字，干活。

一个东西从「我们圈子里玩的东西」变成「大家都可以玩的东西」——这就是 AI Agent 真正走向大众的那一刻。
