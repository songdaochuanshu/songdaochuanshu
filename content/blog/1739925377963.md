**一. custom report type的 UI更新** 

我们在这篇的时候介绍了 custom report type的使用：Salesforce Admin篇(二) Report，可以看到当时的UI很不方便，特别是查看当前的report type的layout是否包含某个字段时，需要一顿找，而且增加lookup关联字段也不是特别的便利。随着现在的release，当前的UI已经做了增强。通过下图我们可以看到，目前可以很方便的查看不同的object都有哪些的字段在layout中，并且很方便的可以增加 section或者增加 lookup 字段并且选择他们对应的字段。

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250217230812337-1187718910.png)

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250217231102223-850695441.png)

 **二. Object Access 一览图**

以前我们想要看Profile 以及Permission Set对于Object的权限，只能一点点查看或者Profile / Permission Set创建 ListView然后查看，现在的系统可以直接在Object中点击Object Access进行查看。

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250218142437162-828748161.png)

 **三. ListView 多字段排序(Beta)**

以前的Listview只能通过一个字段进行排序，现在的环境可以基于最多5个字段进行排序。如何操作？

1\. Setup 搜索 User Interface，然后勾选Enable sort by multiple columns (Beta)

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250218173847148-694189269.png)

 2. 对于你需要操作的ListView，点击Sort by Multiple Columns图标，然后就可以基于自己的需求去排序了。

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250218174043525-454568146.png)

 **四. Conditional Format**

这个功能我们可以对字段基于条件进行设置指定的icon展示，从而可以让 end user快速的查看一些关键字段信息。当前的format只支持icon以及颜色样式，并且只有特定的icon以及颜色，后续官方应该会增强这个功能，放宽自定义icon，颜色或者增加其他的format格式，例如背景色等等，期待后续增强以后的效果。我们以Opportunity的Stage作为demo。我们想要当Opportunity Stage为 Closed won 以及 Closed Lost，显示特殊的icon。

> 注： 此功能目前仅支持 flexipage并且需要启用 dynamic form

 **五. Dynamic Highlight Panel**

升级以前的 highlight panel基于 compact layout assignment，在UI上则显示相关的字段.

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250218200109443-1840138938.png)

 随着最新的升级，我们可以动态的设置 highlight panel，点击Upgrade Now然后进行动态设置，或者在 Fields section进行拖拽，选择你需要展示的字段即可。以前的highlight panel最多可以显示10个字段，dynamic highlight panel最多可以显示12个字段。也可以设置展示哪些action以及什么情况下展示这个 dynamic highlight panel。

![](https://img2024.cnblogs.com/blog/910966/202502/910966-20250218201121404-2057868364.png)
