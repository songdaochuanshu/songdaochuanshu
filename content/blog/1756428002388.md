Dify新版1.8.0发布：新增异步工作流和多模型设置！
============================

你敢相信 Dify 刚刚发布的新版 1.8.0 总共修复并更新了 223 项内容（我手工数的不一定准确，大致就是这个数），简直太炸裂了。

0.新版升级速览
--------

升级的特性有很多，但总体可分为以下三大类：

1.  **新增部分新功能**
2.  **安全性升级**
3.  **性能升级**

1.新功能预览
-------

新增的新功能主要包含以下几个：

1.  **多模型支持**：多环境（开发环境、生成环境）模型独立配置。
2.  **新增工作流变量的默认值**：所有工作流开始节点变量类型现在都支持默认值，使工作流更加健壮。
3.  **MCP 支持 OAuth 登录**：添加了模型上下文协议 （MCP） 支持，用于使用 OAuth 身份验证进行资源发现，从而扩展了集成的可能性。
4.  **代理节点 Token 使用情况监控**：公开代理节点使用指标，以便更好地监控和优化。
5.  **新增知识库文档排序**：在知识库中增加了文档状态的排序功能，改进了文档管理。
6.  可删除头像。
7.  新增管理员自动登录。
8.  Swagger 文档中添加了授权配置支持。

等新功能。

![1](https://img2024.cnblogs.com/blog/172074/202508/172074-20250828154147379-1561393470.png)

2.安全性升级
-------

*   **Redis SSL/TLS 证书身份验证的支持**，以增强安全性。
*   修复 Oracle VDB 中的 SQL 注入。
*   OAuth Redis 兼容性问题和添加了对 Redis SSL/TLS 证书身份验证的支持，以增强安全性。

等安全性升级。

3\. 性能升级
--------

*   **异步工作流升级**：为工作流执行实施异步存储库，显着提高了性能。这种架构变化支持在工作流运行期间进行无阻塞作，早期测试显示典型工作流的执行时间几乎减半。这种优化特别有利于具有多个节点和并行作的复杂工作流程。
*   **Docker 构建优化**：使用 Jest 工作文件的清理脚本优化了 Docker 构建过程。
*   优化数据集清理任务以提高性能。

等性能优化升级。
