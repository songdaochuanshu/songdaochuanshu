推荐一个开箱即用的.NET权限管理平台：Magic.NET
=============================

Magic.NET 是基于 .NET 6/7 + Vue 的开源权限管理平台，前后端分离，开箱即用。集成用户、角色、菜单、文件、定时任务、工作流等企业级功能。模块化设计，代码简洁，适合快速开发 ERP、CRM、OA 等后台系统。Gitee 可获取，免费商用。

大家好，我是刚子。

今天给兄弟们推荐一个我在 Gitee 上发现的开源项目——**Magic.NET**。

如果你正被后台管理系统的增删改查折磨，或者想找个脚手架快速开工，可以看看这个。

一、这是啥？
------

Magic.NET 是一个基于 **.NET 6/7** 的**通用权限管理平台（RBAC模式）**，前后端分离，开箱即用。

开源协议 **Apache-2.0**，免费商用。

二、技术栈
-----

**后端**：.NET 6/7 + Furion + SqlSugar + 多租户 + 动态API + gRPC + SignalR  
**前端**：Vue + Ant Design Vue  
**数据库**：SQLite（默认）、SQL Server、MySQL 随便切

选型挺务实，都是国内生态成熟的东西，拿来就能用。

三、核心功能
------

基本上企业后台该有的都有了：

*   用户/角色/菜单/机构/职位/字典管理
*   应用管理、公告管理、文件管理（支持本地/阿里云/腾讯云）
*   定时任务（cron表达式）
*   日志管理、在线用户监控、服务监控
*   邮件/短信发送（阿里/腾讯）
*   工作流集成
*   SignalR实时通讯

模块化设计，框架升级不影响业务代码。

四、5分钟跑起来
--------

1.  后端：打开 `backend/Magic.sln`，F5运行（默认SQLite）
2.  前端：`frontend` 下执行 `npm install`，再 `npm run serve`
3.  访问：[http://localhost:82](http://localhost:82)

想换数据库？改`dbsettings.json` 里的 `DbType` 和连接字符串就行。  
![dbsettings.json](https://img2024.cnblogs.com/blog/1466017/202604/1466017-20260422170443960-1061804006.png)

五、亮点
----

*   **模块化**：业务层单独模块，框架升级不慌
*   **代码简洁**：不搞过度设计，新手也能看懂
*   **技术新**：.NET 6/7 + Furion + SqlSugar
*   **功能全**：权限、文件、定时任务、工作流、实时通讯全都有
*   **开箱即用**：下载→运行→开工

六、适用场景
------

ERP、CRM、OA、CMS 这类需要权限管理的企业应用。也适合 .NET 初学者学习，或者老手快速搭后台。
