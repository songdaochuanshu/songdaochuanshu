折腾了一下午，终于让Codex用上了DeepSeek
==========================

### Codex简介

AI时代真的风水轮流转，前段时间最火的还是`Claude Code`,转眼间`Codex`就火得一塌糊涂。

`Codex`是由`OpenAI` 推出的AI智能体。

起初呢主要用于编程开发，后面慢慢进化成了通用智能体。界面和操作对于小白来说可谓是非常友好。

它不仅能回答问题，编写代码，还能读取电脑本地文件，修改项目，浏览网页，调用外部工具，自动化执行任务，操作浏览器甚至桌面应用。

![image-20260607092241766](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800628-945607112.png)

也是早早的就给身边不是程序员的亲朋好友安利了，都是用起来就离不开了。

但随着问题也来了——这玩意默认只认 `OpenAI` 自家的模型，得有 `OpenAI` 账号，还得绑定信用卡，又贵又折腾。

我就想：能不能让 `Codex` 走 `DeepSeek`？毕竟 `DeepSeek` 便宜啊，而且国内直连速度快。

结果折腾了一圈，发现直接改配置行不通，得借助中间工具才行。

### 为什么直接改配置走不通？

`Codex` 底层调用的是 `OpenAI` 的 `Responses API`（请求路径是 `/v1/responses`），

而 `DeepSeek` 这些国产模型提供的是 **Chat Completions API**（路径是 `/v1/chat/completions`）。

两套协议的请求格式、流式输出、工具调用方式全都不一样，直接把 `DeepSeek` 的地址填进`Codex`，要么报 404，要么模型列表都刷不出来。

需要一个"翻译官"在中间做协议转换。

### 翻译官来了：CC Switch

**CC Switch** 就是这个翻译官。之前使用过`Claude Code`的朋友应该很熟悉了。

它是一个开源免费的桌面配置工具，专门解决这类 AI 编程工具跟第三方模型对接的问题。

支持 `Codex`、`Claude Code`、`Gemini CLI`、`小龙虾`和`Hemers` 等多款工具，`Windows`、`Mac`、`Linux` 都能用。

### 手把手配置流程

整个过程就四步，熟练的话两三分钟搞定。

### 第三步：开启本地路由

这是最关键的一步，很多人漏了这步导致配完没反应。

点击 CC Switch 右上角的**设置**按钮，进入「路由」页面：

*   **路由总开关**：打开
*   **勾选 CODEX**：确保 Codex 的请求会走本地路由转发
*   **首页打开路由开关**： 回到首页打开路由总开关

![image-20260607084503885](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800600-1283420865.png)

![image-20260607084623869](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800556-2398412.png)

这两个开关打开后，CC Switch 就会在后台启动协议转换代理。代理地址默认是 `http://127.0.0.1:15721`。

回到首页，打开路由总开关

![image-20260607084928189](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800659-1836604806.png)

### 第四步：添加供应商

回到 CC Switch 首页，点击「**添加 供应商**」（右上角的 `+` 号）。

> 这里根据需要可以添加国内各种大模型

![image-20260607085546614](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800684-52817630.png)

在预设列表里选自己喜欢的大模型，这里我选择「`DeepSeek`」。

![image-20260607085421266](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800639-1156228706.png)

CC Switch 会自动帮你填好请求地址，不用手动改。你只需要在「**API Key**」一栏填入你的 \*\*DeepSeek Key \*\*就行。

![image-20260607085725606](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800627-1806685753.png)

Key 去哪弄？

去 **platform.deepseek.com** 注册账号，实名认证后充值（充 10 块能用很久）。

![image-20260607090018971](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800596-1101076393.png)

然后在左侧「API Keys」页面创建一个 Key，复制粘贴过来。**记得提前充值**，余额不足会导致请求报错。

![image-20260607090134351](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800644-107416142.png)

填完 Key 点击保存，然后在供应商列表里对 DeepSeek 点「启用」。

![image-20260607090214364](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800647-1836357286.png)

### 第五步：重启 Codex 验证

如果 Codex 正在运行，**先完全退出**，再重新打开。配置变更后必须重启才生效。

![image-20260607090426520](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800621-1308377329.png)

打开后看界面**左上角**，如果模型/Provider 显示的是 **DeepSeek**，说明路由和供应商配置都已经生效了。

在对话框里输入"你好，你用的是说明模型"测试一下，能正常收到回复就是接通了。

![image-20260607090540458](https://img2024.cnblogs.com/blog/2381533/202606/2381533-20260609075800664-897162696.png)

### 整个链路长什么样

配完之后的请求流转是这样的：

![原理图](https://blog.xiezhrspace.cn/blog-img/%E5%8E%9F%E7%90%86%E5%9B%BE.png)

`Codex` 始终以为自己在跟 `OpenAI` 对话，实际上 `CC Switch` 在中间当"翻译"，把 `Responses API` 的请求转成 `Chat Completions` 格式发给 `DeepSeek`，再把响应包装回 `Codex` 认识的格式。两边都觉得自己在正常说话。

### 用起来怎么样

*   **速度**：国内直连 `DeepSeek`，不用走代理，响应比连 `OpenAI` 快不少
*   **费用**：`DeepSeek` 的价格大概是 `GPT-4o` 的十分之一，写代码这种高频场景能省不少钱
*   **模型选择**：`CC Switch` 里除了 `DeepSeek`，还能接 `Kimi`、`智谱 GLM`、`XiaoMi MiMo`等，点一下就能切
*   **数据安全**：代码和请求都在国内流转，不用往海外传

而且 `CC Switch` 不只管 `Codex`，`Claude Code`、`Gemini CLI` 这些工具也能统一管理，一个面板管所有工具的模型和 Key，比自己手动改配置文件省心太多。

### 常见问题

**Codex 一直卡在启动界面？**  
检查网络是否能正常访问 `OpenAI / Codex` 服务，首次启动需要这个。完全退出 Codex 进程再重新打开试试。

**CC Switch 里 DeepSeek 已启用，Codex 还是显示旧模型？**  
确认 `CC Switch` 首页的路由开关已打开，且 CODEX 项已勾选。然后完全退出 Codex 重启。

**请求报错或无响应？**  
检查 `DeepSeek API Key` 是否有效、账户是否已充值。在 CC Switch 里看 `DeepSeek` 供应商状态是不是"运行中"。

如果你也在用 Codex 但不想绑 OpenAI 的 API Key，或者想省点模型调用的费用，这套方案值得试试。

整个过程不需要复杂配置，全是图形界面点几下的事。
