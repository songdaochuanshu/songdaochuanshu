Rudist v0.5.1 发布：AI 驱动的 Redis 客户端，更快、更直观
========================================

> 如果你每天都在和 Redis 打交道，你一定有这种时刻：记不住 KEYS 和 SCAN 的语法、在 MEMORY STATS 里找半天、或者小心翼翼地执行一条可能有副作用的命令。  
> Rudist 的目标，就是让这一切变成"说一句人话"。

今天，Rudist 正式发布 **v0.5.1**，带来了全新的实时日志面板、更流畅的 AI 交互，以及更丰富的快捷键体系。  
![rudist-demo](https://img2024.cnblogs.com/blog/2157887/202604/2157887-20260414161218059-1455200310.gif)

* * *

AI 能力：从"自然语言"到"Redis 命令"
========================

这是 Rudist 最核心的能力。

不再需要记忆复杂语法，只需在输入框里说一句话，AI 就能理解你的意图并生成对应的 Redis 操作：

![image](https://img2024.cnblogs.com/blog/2157887/202604/2157887-20260414154606321-1120777468.png)

18+ AI 工具，涵盖真实 Redis 工作流
------------------------

Rudist 的 AI 不只是"翻译"成命令，它深度集成了 Redis 操作上下文，知道你的数据结构、TTL、内存占用，能够：

工具

功能

`scan_keys`

按模式扫描 key，支持通配符

`read_string` / `read_hash` / `read_list`

读取多种类型的数据

`get_ttl` / `get_memory`

查看 TTL 和内存占用

`write_command`

执行任意 Redis 命令

`get_info`

获取 Redis INFO

`analyze_data`

分析数据模式，给出优化建议

`explain_unusual`

解释异常数据或配置

`suggest_cleanup`

智能建议清理策略

...

还有更多

多模型支持，一个界面兼容所有
--------------

Rudist 支持 OpenAI、Anthropic Claude、Ollama（本地部署）、OpenRouter 等多种模型，多达26个供应商支持。  
API Key 安全存储在系统 Keychain 中，不会在任何配置文件中明文出现。

快捷键优先：键盘就是你的武器
==============

这是 Rudist 与其他 Redis GUI 客户端最不一样的地方——**快捷键不是辅助，而是默认操作方式**。

v0.5.1 中，快捷键体系进一步丰富：

### 核心快捷键

快捷键

功能

`Cmd/Ctrl + E`

**打开 AI 面板** ← 从这里开始

`Cmd/Ctrl + L`

**切换实时日志面板**

`Cmd/Ctrl + R`

刷新当前 Key 的值

`Cmd/Ctrl + F`

聚焦 Key 过滤器

`Cmd/Ctrl + N`

新建连接

`Cmd/Ctrl + Enter`

确认新建连接

`F5`

刷新 Key 列表

`↑ / ↓`

浏览历史命令

### Tab 与连接管理

快捷键

功能

`Cmd/Ctrl + 1-9`

切换到 Tab 1-9

`Cmd/Ctrl + 0`

切换到上一个 Tab

`1-9`（无修饰键）

快速连接保存的连接 1-9

`0`（无修饰键）

重连所有未关闭的连接

### AI 交互专属

快捷键

功能

`→`（右箭头）

执行 AI 建议的命令

`←`（左箭头）

取消确认对话框

`Esc`

关闭 CLI / 取消操作

所有快捷键均可自定义（Tab 切换、快速连接等少数固定快捷键除外）。

实时日志（Live Logs）：v0.5.1 新功能
==========================

**这是本次版本最重要的新功能。**

当你发送一条 AI 命令时，背后发生了什么？Live Logs 给你答案。

![image](https://img2024.cnblogs.com/blog/2157887/202604/2157887-20260414155352907-494208557.png)

*   **按 `Cmd/Ctrl + L` 随时呼出/隐藏**日志面板
*   实时显示 AI 命令的完整执行过程
*   包括 Redis 原始响应、错误信息、执行时间
*   同一面板同时记录 AI Chat 和 Redis 执行两条日志流

对于调试 AI 命令行为、理解 AI 生成的 Redis 操作来说，这个面板比任何日志文件都直观。

安装与上手
=====

#### 从 GitHub Releases 下载

[https://github.com/your-repo/rudist/releases/tag/v0.5.1](https://github.com/your-repo/rudist/releases/tag/v0.5.1)

### 首次使用

1.  打开 Rudist，Cmd/Ctrl + N添加一个 Redis 连接（支持连接字符串直接导入）
2.  按 Cmd/Ctrl + E 打开 AI 面板
3.  输入你的问题，享受 AI 带来的效率提升
