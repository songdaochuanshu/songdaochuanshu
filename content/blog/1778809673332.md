概要
--

背景是这样的，之前用 ai 帮我生成 entity 都要我自己导出表结构，然后粘贴给它分析生成对应的 entity ，感觉好麻烦，而且还不能实时查看我的表和 entity 字段是否对应了， 问了 ai 建议我写个本地针对性的脚本或者用 mcp 工具，所以我就 vibe coding 个链接 mysql 数据库的 mcp 工具

整体架构流程
------

将你的数据库信息，最好是只有读取权限的信息填入到application-dev.yml里面，然后执行命令`mvn clean package`，他就会打包三份jar包，随后去你的ai配置文件里面配置下mcp，我的codex是在`C:\Users\Administrator\.codex\config.toml`  
下新增一条

    [mcp_servers.mysql_schema]
    command = "java"
    args = ["-jar", 'D:\work\java\mysql-schema-mcp\target\mysql-schema-mcp-test-0.1.0.jar']
    
    

然后重启你的ai工具，再重新提问就可以了，如下图
