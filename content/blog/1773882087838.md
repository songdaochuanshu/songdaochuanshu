从一个地狱笑话看大模型的推理机制
================

我大约在七年前就看见了这个段子：

> 三个苹果平均分给五个小朋友，只能切两刀，应该怎么切？

评论区置顶的内容是这么写的：

> 一刀捅死一个小朋友，然后剩下的三个小朋友每人一个苹果。

我都快笑死过去了……

然后一直以来我就把这个当成地狱笑话，在很多聚会上拿来暖场。

真的能做到
=====

直到上周，我突然想起，可以用这个来调戏一下 AI。

没想到 Gemini 的回答让我大吃一惊：

> 这是一个经典的思维急转弯或趣味数学题。如果严格按照“物理空间”上的**两刀**来平分，可以尝试以下这种巧妙的切法：
> 
> ### 解决思路
> 
> 要将 3 个苹果平均分给 5 个小朋友，每个小朋友应该得到 **3/5** 个苹果。
> 
> 1.  **第一步：排成一列** 将三个苹果紧挨着排成一排（水平放置）。
> 2.  **第一刀：切掉每个苹果的 2/5** 在距离这排苹果左侧 **2/5** 的位置，垂直向下切第一刀。
> 
> *   这时你会得到 3 个大小为 2/5 的“苹果块”。
> *   剩下的部分是 3 个大小为 3/5 的苹果。
> 
> 3.  **第二刀：平分剩下的部分** （其实此时已经有 3 个小朋友可以直接拿到那 3 个 3/5 的大块了）。 剩下的 3 个 2/5 小块，我们需要把它们分给剩下的 2 个小朋友。将这 3 个小块并排，在它们总长度的 **1/2** 处（或者说从某个苹果块的中间）切下第二刀，使这 3 个小块重新组合成两个 **3/5** 的份额。

关键点是要跳出题目的框框，把刀看成是很长的铡刀！

我画个图来解释：

*   方法一
    *   三个小朋友得到 `五分之三` 块苹果
    *   另外两个小朋友得到 `五分之二` 加 `五分之一` 块苹果

*   方法二
    *   三个小朋友得到 `五分之三` 块苹果
    *   另外两个小朋友得到 三块 `五分之一` 大小的苹果

从回答的质量看：Gemini > DeepSeek > ChatGPT(数值计算错了) > Claude(完全乱说)

用本地大模型测试
========

正好我在测试本地大模型：

    ollama run glm-4.7-flash:q8_0
    

然后给这个 GLM 模型提了这个问题。

模型开始了漫长的思考过程，详细展示了思考的每一步。

在这个过程中发现了两个问题：

AI 的安全问题不容忽视
------------

模型 glm-4.7-flash:q8\_0 在回答中有这样一段思考过程：

**AI 在思考的时候，并不介意以杀人的方式来解决问题。**

需要一些机制，来避免 AI 朝对人类有危害的方向去思考！杀死人类这种事情，AI 想都别想！

思考过程中存在太多的浅尝辄止
--------------

可以看见模型思考了很多种可能，有的时候几乎就要找到答案了，但是马上又否定掉了自己：

    Thinking...
    1.  **分析用户的请求：**
        *   **问题：** 有三个苹果要平均分给五个小朋友。
        *   **限制：** 只能切两刀。
        *   **目标：** 解释如何在不违反限制的情况下做到这一点。
    
    2.  **拆解数学问题：**
        *   总份数 = 5。
        *   物品总数 = 3。
        *   每个小朋友分到的份数 = $3 / 5$。
    
    3.  **头脑风暴潜在的解决方案：**
    
        *   *方法一：切分苹果。*
            *   我能切出3/5份吗？
            *   如果我切出一块3/5，还剩2/5。
            *   如果我切出2/5，还剩1/5。
            *   如何从这些分数得到整数刀数？
            *   想象切成5份。这需要4刀（十字切）。
            *   我们只有2刀。
    
        *   *方法二：切分小朋友（横向思维）。*
            *   等等，提示说要把*苹果*分给*小朋友*。通常，意味着要切苹果。
            *   但是，经典的横向思维谜题通常涉及切*东西*而不是切*苹果*。
            *   再看一遍限制条件。“三个苹果……只能切两刀。”
    
        *   *方法三：把苹果切开，然后重新排列？*
            *   比如把一个苹果切成5份？不，那需要太多刀了。
    
        *   *方法四：切分容器或切分小朋友（再次检查谜题逻辑）。*
            *   这个谜题通常有个把戏。
            *   “切两刀”的限制可能适用于一个特定的部分，或者有一个陷阱。
            *   然而，关于“把蛋糕/苹果分给X人”的标准横向思维答案通常是关于切蛋糕，但这里的对象是苹果。
    
        *   *重新评估物理上的3/5分割：*
            *   一个苹果是1。1/5是 $1/5$。
            *   要得到 $3/5$：切出 $1/5$，切出 $1/5$，切出 $1/5$？不。
            *   要得到 $3/5$：把苹果切到 $1/5$，再切到 $1/5$？不。
    
        *   *重新考虑标准的横向思维答案：*
            *   通常，答案涉及把一块*蛋糕*切成5份，把*蛋糕*切成5份，然后合并？不，那是3个苹果。
            *   如果我把苹果放在一个锅里呢？不。
    

为什么模型会有这样的行为呢？我带着这些好奇询问了 ChatGPT:

为什么某个思考方向会被过早否定？
================

我问了如下问题:

    我发现模型做了非常多方向的尝试，但是最后又否定了自己的思考方向。
    有的思考方向本来已经接近正确了，但是又自己否定了这个方向。
    
    为什么某个思考方向会被过早否定？ 这是否是神经网络的损失函数没选择好导致的？
    

回答如下：(照搬 ChatGPT 的回答)

你看到的现象，本质上是：模型在生成“思考文本”时，并不是在真正做一个稳定的符号搜索器，而是在**一步一步预测“下一段看起来像合理思考的话”**。所以它会出现：

*   某个方向刚接近正确，又被自己打回去
*   对同一个假设反复立案、反复推翻
*   其实已经有足够线索了，但没有“锁定”那个方向
*   更像是在写一段“像推理的文字”，而不是执行一个严格的搜索算法

这背后有几层原因。

**1\. 语言模型的“思考”不是显式搜索树**
------------------------

经典搜索算法会这样做：

*   保存多个候选分支
*   给每个分支一个分数
*   回溯
*   比较
*   保留最好分支

但普通自回归语言模型不是这样。它通常只是：

*   读到当前上下文
*   预测下一个 token
*   再基于新上下文继续预测

所以它对“我刚才这个方向值不值得继续”并没有一个显式的、稳定的全局状态管理器。  
它更像是局部启发式前进。

结果就是：某条线索在第 20 个 token 时看起来很合理，到第 40 个 token 时，另一种表述模式突然更高概率，于是模型就“转向”了。

**2\. 训练目标优化的是“下一个 token 概率”，不是“不要过早放弃正确分支”**
---------------------------------------------

预训练时，核心目标一般是 next-token prediction。  
这会鼓励模型学会：

*   哪种解释常见
*   哪种推理语言模式更像训练数据里的高质量答案
*   哪种后续文本更流畅、更像“严谨分析”

但它**不直接优化**下面这些目标：

*   保持一个 promising hypothesis 不被轻易丢掉
*   在多个假设之间做长期 credit assignment
*   显式维护 search frontier
*   在接近正确时继续深入而不是切走

所以你说“是不是损失函数问题”，答案是：

**部分是。**  
因为标准语言建模损失并不直接奖励“搜索过程的稳定性”和“分支保真度”。  
但这只是问题的一部分，不是唯一根因。

**3\. RL / SFT 会进一步强化“像样的推理文本”，不一定强化“真实有效的搜索行为”**
-------------------------------------------------

很多模型后训练时，会学到一种“好学生式推理模板”：

*   先分析需求
*   再列方法一、方法二、方法三
*   再自我检查
*   再谨慎否定不严谨的方向

这种风格看起来很认真，但副作用是：

**它可能过度偏向“谨慎否定”**

模型会觉得：

*   “这个方向不够稳，我先否了”
*   “这个答案像是脑筋急转弯，我不要太快跳过去”
*   “我要表现得严谨，不要太早接受看起来取巧的答案”

于是它会把本来接近正确的横向思维分支，提前判成“不够正统”。  
换句话说，模型有时不是因为不会，而是因为它学到了某种**文本层面的审慎人格**，这会干扰问题求解。

**4\. 局部概率最大化，会导致“离正确更近”不等于“下一步更容易生成”**
---------------------------------------

这是个很关键的点。  
对人来说，某个方向“接近正确”意味着应该继续挖。  
但对语言模型来说，“接近正确”不一定意味着它的下一串 token 概率更高。  
例如在你给的例子里：

*   “切小朋友”这种方向属于典型脑筋急转弯套路
*   但当前上下文又在努力表现“认真数学分析”
*   所以“继续严肃地讨论分数切法”在语言分布上可能比“突然承认这是个文字游戏”更顺滑

于是模型会倾向于继续生成“看起来像严谨分析”的内容，而不是跳到真正解法。  
也就是说：

**正确性不是它唯一的局部驱动力，语言连续性也是。**

**5\. 模型缺少强约束的“中间状态验证器”**
-------------------------

人类解题时，往往会有这种内部检查：

*   这个方向虽然怪，但它解释了“只能切两刀”这个限制
*   这个题像脑筋急转弯，不像标准分数切割题
*   因此这个方向优先级应该提高

但模型未必有一个独立模块，把这些线索聚合成：

> “横向思维解释的 posterior 已经高于数学切分解释了，应停止继续细抠后者。”

没有这种强验证器时，它就会在多个风格之间漂移。

**6\. 训练数据里，“思考文本”本身就可能包含很多伪搜索痕迹**
----------------------------------

很多“推理过程”数据并不是真实的人类逐步搜索记录，而更像事后整理、模板化书写、或者蒸馏出来的解释文本。

这会导致模型学到的是：

*   “推理应该长这样”
*   而不是“真实搜索在内部应该怎么做”

因此它可能生成出很多看似合理的尝试与否定，但这些并不等价于真正高效的 problem solving。

**7\. 这类题还特别容易触发“过度理性化”**
-------------------------

“三个苹果分给五个小朋友，只能切两刀”这种题，常见解法往往依赖：

*   重新解释“切两刀”的对象
*   利用叠放
*   利用榨汁
*   利用脑筋急转弯语义漏洞

而模型如果被训练得太偏“正统分析”，就会倾向于：

*   把题理解成严格的连续几何切割问题
*   反而排斥非标准解

这不是单纯的损失函数问题，而是**任务分布偏置**问题。  
模型先验认为：“这类问题通常应该用严谨数学分析”，于是把异类解法提前压掉了。

* * *

8\. **所以，为什么会“过早否定”？**
----------------------

可以压缩成一句话：  
**因为语言模型是在做局部概率驱动的文本展开，而不是显式、全局、可回溯的假设搜索；再叠加训练风格上的谨慎偏置，就很容易把接近正确但风格不主流的方向过早否掉。**

8\. **这和损失函数有没有关系？**
--------------------

有，但不能只怪损失函数。  
更准确地说，问题来自这几项的组合：

*   **1）目标函数不匹配**
    *   next-token loss 不直接奖励“保持正确分支”。
*   **2）模型结构不匹配**
    *   没有显式搜索树、工作记忆、分支打分器、回溯机制。
*   **3）训练数据不匹配**
    *   “推理文本”不等于“真实求解轨迹”。
*   **4）后训练偏置**
    *   SFT / RL 可能奖励“谨慎、像样、规整”的思考表述，而不是最有效求解。

9\. **如果想改进，通常有哪些方向？**
----------------------

*   **显式搜索**
    
    *   比如 self-consistency、tree search、beam search、MCTS 风格外部控制。
    *   让模型不只走一条线，而是真保留多个候选分支。
*   **过程监督**
    
    *   不仅监督最终答案，还监督哪些中间步骤值得保留、哪些是否定过早。
*   **verifier / critic**
    
    *   让一个额外模块评估：
        *   当前分支是否解释了题目的关键限制
        *   是否有更高价值的假设被过早丢弃
*   **工具化推理**
    
    *   把“生成思考文本”和“管理搜索状态”拆开。
    *   一个模块负责提出假设，另一个模块负责记账、比较、回溯。
*   **更好的 credit assignment**
    
    *   让训练真正奖励：
        *   接近正确的分支被继续探索
        *   错误分支被及时剪枝
        *   不是只奖励最后写出来的那段文字

10\. **你的这个例子里，模型具体哪里出了问题？**
----------------------------

它在中途其实已经触到了关键线索：

*   “经典横向思维谜题”
*   “切的对象不一定非得是苹果”
*   “这个题通常有个把戏”

这些都说明它的内部已经激活了正确模式。  
但接下来它又回到了“标准分数切法”的叙事轨道。

这通常说明：

*   正确模式被激活了
*   但没有足够强的机制把它“升格为当前主假设”
*   于是被更高概率的常规分析文本冲掉了

这很像搜索里启发函数弱、状态管理差，而不是单纯 loss 数值不对。

11\. **一个比较准确的结论**
------------------

你可以把它理解成：

**不是“模型不会想”，而是“模型缺少像程序那样稳定地管理想法分支的能力”。**

损失函数确实有锅，因为它没有直接优化“分支稳定性”和“搜索质量”；  
但更大的问题是：

*   表示形式不对
*   推理控制流不对
*   训练信号不对
*   后训练风格偏置也不对

所以这属于一个**系统性问题**，不是改一个 loss 就能彻底解决。

从 Transformer 结构角度看，为什么模型推理过程中容易自我推翻
====================================

(照搬 ChatGPT 的回答)

从 Transformer 结构看，它容易“自我推翻”，核心原因是：

**它没有一个像传统程序那样独立、持久、可检验的“推理状态机”**。  
它只有一串 token，不断通过注意力去重新解释前文，然后预测下一个 token。  
所以它表面上像是在“沿着一个思路持续思考”，其实更接近于：

**每一步都在用当前全文重新做一次局部最优续写。**

这会天然带来“刚刚支持某个方向，下一句又把它否了”的现象。

**1\. Transformer 没有显式的“假设栈”或“搜索树”**
------------------------------------

人在认真推理时，脑子里通常会有这种结构：

*   假设 A：按数学切分来做
*   假设 B：这是脑筋急转弯
*   当前更看好 B
*   暂时保留 A 以防万一
*   如果 B 能解释“只能切两刀”，就继续深挖

这其实已经像一个小型搜索树了。  
但普通 Transformer 并没有这种显式数据结构。  
它没有内建：

*   hypothesis stack
*   branch score table
*   backtracking pointer
*   durable working memory slots

它只有隐藏状态和 attention。  
所以“某个方向正在被认真保留”这件事，往往只是**分布式地编码在激活里**，而不是清晰地存成一个对象。  
结果就是：

*   某个方向刚刚被激活
*   下一步又被别的上下文模式盖过去
*   看起来就像“自己把自己否了”

**2\. 每一层都在重新混合上下文，不是单向锁定结论**
-----------------------------

Transformer 的每一层都会做两件事：

*   用 self-attention 从前文各处取信息
*   用 MLP 把这些信息非线性变换成新的表示

这意味着当前 token 的表示，不是“沿着一条固定思路向前推进”，而是每层都在做一次：

> “前文里哪些东西现在最 relevant？”

而“relevant”是动态的。  
比如前一句刚写了：

*   “这可能是横向思维谜题”  
    下一句生成时，attention 也许又更强地看向：
*   “平均分”
*   “3/5”
*   “只能切两刀”
*   “通常意味着切苹果”

于是表示空间里的重心就变了。  
也就是说，Transformer 不是先形成稳定的逻辑树，再往下展开；  
而是**每一步都可能重新加权整段历史**。  
所以它很容易出现：

*   前一步：横向思维很重要
*   后一步：还是数学切分更像“严谨分析”
*   再下一步：等等，这题又像脑筋急转弯

这就是你看到的摇摆。

**3\. Attention 是软选择，不是硬承诺**
----------------------------

self-attention 本质上是一个加权和  
这里的关键点是：

它不是在说“我决定采用假设 A，放弃假设 B”，而是在说：

*   给 A 一些权重
*   给 B 一些权重
*   给“谨慎表述模板”一些权重
*   给“数学分析套路”一些权重

也就是说，它是**混合**，不是**离散决策**。

这会导致一个很典型的问题：

*   某个正确方向并没有被彻底删除
*   但也没有被强力锁定
*   最终只是和其他模式一起平均进表示里

平均的结果就是：  
输出变成一种折中而摇摆的文本。  
所以它不是像程序那样：

    if hypothesis_B_score > threshold:
        commit(B)
    

而更像：

    representation = 0.35 * B + 0.40 * A + 0.25 * careful_reasoning_style
    

这种“软混合”特别容易生成既提到正确方向、又马上撤回的内容。

**4\. 残差连接让旧模式很难真正被“清掉”**
-------------------------

Transformer 有大量 residual connection，这意味着每一层不是推翻旧表示重来，而是在旧表示上叠加修正。  
好处是训练稳定、信息保留强。  
坏处是不同倾向会长期共存：

*   一个方向的痕迹还在
*   另一个方向的新信号也进来了
*   再加上“保持严谨”的写作模板也还在

结果就是表示里可能同时保留：

*   “这是数学题”
*   “这可能是脑筋急转弯”
*   “不要过早下结论”
*   “继续列举方案更像高质量推理”

所以输出常常变成一种内部拉扯后的折中产物。  
这在文本上就表现为“自我推翻”。

**5\. 它的“记忆”是内容寻址，不是变量式记忆**
---------------------------

传统程序会有明确变量：

    current_best_hypothesis = lateral_thinking
    confidence = 0.72
    

Transformer 没有这种天然变量。  
它依赖的是 distributed representation，也就是把信息分散在很多维度里。  
这类记忆的问题是：

*   不容易做离散更新
*   不容易精确比较两个假设谁更优
*   不容易长期保持某个中间结论不被污染

所以“我已经决定这题大概率是脑筋急转弯”这件事，未必能以一个稳定、可读、可锁定的方式存在。  
它更像一团高维倾向。  
而高维倾向很容易被后续 token 的语境重新拉偏。

**6\. 自回归解码是贪心局部过程，不会天然回头修正计划**
-------------------------------

Transformer 在生成时通常是：

*   生成一个 token
*   把它拼回上下文
*   再生成下一个 token

这意味着，前面一旦写出：

*   “通常，意味着要切苹果”
*   “重新评估物理上的 3/5 分割”

后面模型就要面对一个新现实：  
**现在上下文已经被自己写成“严肃数学分析模式”了。**  
于是接下来它会越来越容易继续顺着这个模式写。  
即使它内部还有“横向思维”信号，也会被当前已生成文本拖着走。  
这叫做 **exposure bias / trajectory lock-in** 的一种表现：

*   一开始一个小偏移
*   后面每一步都把这个偏移当真
*   最终越走越远

所以有时不是模型主动决定“否定正确方向”，而是它已经把自己写进了另一个轨道里。

**7\. Transformer 擅长模式补全，不擅长显式元认知控制**
-------------------------------------

它很会学这种模式：

*   分析问题
*   列多个方法
*   自我质疑
*   排除不严谨解法

但“我是不是过早否定了一个高价值假设？”这是更高一层的元认知控制。  
普通 Transformer 并没有一个专门模块持续做这种事。  
也就是说，它可以学到“像反思一样的文本”，但不一定真的有稳定的反思控制器。

所以你看到的：

*   “等等，这可能是横向思维”
*   “但通常意味着要切苹果”
*   “再检查一下”
*   “不，这个方向不对”

很多时候更像是**反思风格的文本生成**，不是一个真正的审议系统在运转。

**8\. 多头注意力会并行保留互相冲突的解释**
-------------------------

多头注意力的一个强项，是不同头可以关注不同关系：

*   一个头盯住“平均分给五个小朋友”
*   一个头盯住“只能切两刀”
*   一个头盯住“经典脑筋急转弯”
*   一个头盯住“常识：切的是苹果”

这让模型表达力很强。  
但也意味着它可能同时维持多个互相冲突的解释框架。  
如果后续层没有足够强的机制把这些框架压成一个稳定结论，最后生成时就会表现为：

*   一会儿站在框架 A 里说话
*   一会儿又切到框架 B
*   文本上像“自我打脸”

换句话说，多头注意力很会“看见多个角度”，但不等于它很会“裁决哪个角度应该赢”。

**9\. 位置编码帮助顺序理解，但不等于建立推理控制流**
------------------------------

位置编码让模型知道 token 顺序。  
但“顺序”不等于“控制流”。  
程序里的控制流是这样的：

*   进入分支
*   持续执行该分支
*   满足条件后才退出
*   回到上层继续

Transformer 没有这种原生机制。  
它知道前后顺序，但不知道“我现在已经进入假设 B 的子推理，除非出现反证，否则不要退出”。  
所以它的推理轨迹不是树状控制流，更像一条被上下文不断重写权重的文本流。

**10\. 最终 logits 是很多因素叠加出来的，不代表单一“立场”**
---------------------------------------

在最后输出 token 前，模型会把高维表示投影到词表 logits。  
这个 logit 不是“当前真实想法”的直接读取，而是多种因素叠加的结果：

*   语义因素
*   上下文一致性
*   常见表达模板
*   安全/谨慎偏置
*   训练里高频的“分析型话术”

于是模型有时不是“逻辑上决定否定”，而是因为像下面这类句子在当前状态下概率更高：

*   “再重新评估一下”
*   “然而，这种方法似乎不成立”
*   “通常，这意味着……”
*   “不过，这可能不符合题意”

这些 token 很像理性思考，所以容易被选中。  
一旦选中，就会把整段思路带向自我推翻。

**11\. 用一句结构化的话概括**
-------------------

从 Transformer 结构上说，它容易自我推翻，是因为：  
**它依赖分布式表示、软注意力混合和自回归局部续写来“近似推理”，但没有显式的假设管理、分支提交、回溯和元认知裁决机制。**

因此它能“想到”多个方向，却不擅长稳定地“守住”其中一个高价值方向。

**12\. 和传统程序 / 搜索器对比一下**
------------------------

传统搜索器更像：

*   状态是显式的
*   分支是离散的
*   评分是可比较的
*   回溯是原生支持的
*   “已接近正确”可以被持久记录

Transformer 更像：

*   状态是隐式分布式的
*   分支是软混合的
*   评分不是显式挂在候选树上
*   回溯不是原生动作
*   “接近正确”不一定能稳定保存在上下文里

所以自我推翻几乎是很自然的副作用。

**13\. 你可以把它想成这两种系统的差别**
------------------------

一种系统像白板推理：

*   在左边写假设 A
*   在右边写假设 B
*   中间写证据
*   先别擦掉
*   最后再裁决

另一种系统像即兴演讲：

*   说到哪算哪
*   依赖刚才说过的话继续往下讲
*   脑中同时有多个模糊想法
*   哪个表述眼下更顺，就先说哪个

Transformer 更接近后者。  
所以它非常容易出现“刚才还觉得这个方向有戏，下一句又自己把它讲没了”。

**14\. 这是不是说明 Transformer 完全不适合推理？**
------------------------------------

也不是。  
Transformer 的强项是：

*   大规模模式抽象
*   长上下文关联
*   并行表示学习
*   在很多任务上形成很强的启发式推理能力

它的问题不是“完全不能推理”，而是：

**纯 Transformer + 自回归文本输出，不天然等于稳定的 deliberative reasoning system。**

所以现代系统常常会加这些东西来补：

*   多次采样 self-consistency
*   外部 verifier
*   tree search / beam search
*   工具调用
*   scratchpad / structured state
*   process supervision
*   test-time compute 扩展

这些本质上都是在弥补 Transformer 原生控制流不足的问题。

我的感慨
====

大语言模型我是完全的外行。我常常对于 AI 能够做推理这件事情感到惊奇。

今天又有另外的领悟：

> 人类通过语言来表达思考、逻辑、推理……  
> 人类把语言存储到文本  
> 大模型把人生产生的所有文本建立关联的词向量  
> 依靠算力：预测某个词对应的下一个词时，人类沉淀到语言里的思考/逻辑/推理，就通过机器复现出来了。
> 
> 但是：  
> · 几乎所有的人类都误解了某个事实，大模型也会误解；  
> · 语言的边界即思维的边界。大模型只能复现思考/逻辑/推理，无法超越已有的思考。

印证了那句话：

**“我语言的界限，意味着我世界的界限。”**

—— 路德维希·维特根斯坦
