极速安装体验openClaw
==============

🚀 OpenClaw 极速安装与全流程避坑指南：从零到完美运行
================================

OpenClaw 凭借其强大的 AI 自动化能力最近非常火爆。但在 Windows 环境下安装，如果不注意细节，很容易在“第一步”就卡死。本文总结了一套从**底层环境搭建**到**后台服务运行**的极速流程，适合所有想体验 OpenClaw 的朋友。

* * *

🛠️ 一、 安装前的“地基”准备（必看！）
----------------------

很多朋友安装报错 `ENOENT` 或 `Permission denied`，其实是基础环境没配好。

### 1\. 安装 Node.js (推荐使用 nvm)

OpenClaw 需 Node.js >= 22 版本。Windows 用户推荐使用 **nvm** 管理 node 版本，这样切换和管理更方便。  
  
安装成功后请使用 **管理员权限** 打开 PowerShell，输入 `nvm -v` 验证。  

### 2\. 必须安装 Git

OpenClaw 许多组件需要 Git 拉取。输入 `git --version` 验证，没装请先去官网下载。  

### 3\. 配置镜像源与解决认证问题

国内环境下，设置 npm 镜像并强制 Git 走 HTTPS（避开 SSH 认证报错）是极速安装的关键：

    # 设置 npm 镜像
    npm config set registry https://registry.npmmirror.com
    # 强制 Git 走 HTTPS 协议
    git config --global url."https://github.com/".insteadOf git@github.com:
    

* * *

📦 二、 极速安装步骤
------------

推荐使用 `npm` 全局安装，这是最省心的方式：

    # 如果之前装过但报错，建议先清缓存
    npm cache clean --force
    # 全局安装
    npm install -g openclaw@latest
    

安装完成后，输入 `openclaw --version` 看到版本号即代表本体安装成功。

* * *

🛡️ 三、 初始化配置（避坑精华）
------------------

**这是最关键的一步！** 必须解决“权限”和“认证”问题。

### 1\. 以管理员身份运行 PowerShell

**右键选择“以管理员身份运行”**。否则会导致后台服务安装失败，网页控制台打不开。

### 2\. 执行引导指令

    openclaw onboard --install-daemon
    

  
一路选择 `Yes` 和 `QuickStart`。  

### 3\. Kimi 用户划重点：Key 选哪个？

  
如果你用的是 Kimi，这一步千万别选错：

*   **Auth Method**：务必选 **`Kimi Code API key (subscription)`**（对应 Kimi Code 订阅页面生成的 Key）。  
    
*   **输入 Key**：粘贴你获取到的 API Key。  
    
*   **Model**：推荐选 **`kimi-coding/k2p5`**，这是目前的旗舰模型，响应最快。  
    

### 4\. 关于 Telegram 和其他配置

  
由于网络环境原因，Telegram 建议先跳过。等页面能开了再在网页端配置代理，否则配置过程中会一直报网络错误。

* * *

🌐 四、 启动与访问
-----------

完成配置后，系统会自动安装后台服务（Daemon）。

*   **访问地址**：`http://127.0.0.1:18789/`  
    
*   **验证运行**：关掉终端窗口，如果网页依然能正常聊天，说明后台服务安装成功！  
      
    

* * *

⌨️ 五、 OpenClaw 常用指令大全
---------------------

熟练使用这些指令，能帮你解决 99% 的问题：

### 1\. 服务管理 (Daemon)

*   `openclaw daemon status`：查看后台服务是否正在运行。
*   `openclaw daemon start`：手动启动后台服务。
*   `openclaw daemon stop`：停止后台服务。
*   `openclaw daemon restart`：重启服务（修改配置后常用）。

### 2\. 状态与诊断

*   `openclaw status`：查看当前配置的所有通道和状态。
*   `openclaw doctor --fix`：**【强力推荐】** 自动扫描并修复不合法的配置文件。

### 3\. 配置管理

*   `openclaw onboard`：想换模型、换 Key 或重新初始化时使用。

* * *

### 💡 结语

只要搞定了 **Git 环境** 和 **管理员权限**，OpenClaw 的体验非常丝滑。希望这份指南能帮你快速通关，开启你的 AI 自动化之旅！
