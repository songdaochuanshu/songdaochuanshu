一、这到底是个什么神仙项目？
--------------

简单说，这是一个​**专门为 MicroPython 内核开发者量身打造的 Claude 插件集合**​。

Claude Code 是 Anthropic 推出的 AI 编辑器，支持插件扩展；而这个项目，把从「代码规范、本地 CI、PR 管理到设备交互」的所有繁琐流程，都做成了 Claude 能直接调用的「对话式技能」—— 你不用离开编辑器，不用记复杂命令，只要跟 AI 说一句话，就能搞定所有环节。

二、每个插件都精准戳中你的开发痛点
-----------------

我把所有插件拆成了「规范 - 测试 - PR - 调试」四大模块，每个都解决真实的开发痛点：

### `mpy-rules`：你的专属 MicroPython 规范「贴身助教」

你是不是也这样？\*\*每次写代码都要翻官方 `CODESTANDARDS.md`，记不住缩进、命名、架构约定，提交 PR 还被上游打回说「不符合规范」？

这个插件会**自动加载 MicroPython 所有开发规则**到你的 Claude 会话里：

*   核心构建 / 测试规范
*   整体架构指南
*   PR 工作流要求  
    而且是通过「会话启动钩子」自动生效 —— 你不用手动复制任何配置文件，打开项目的那一刻，Claude 就已经懂了所有 MicroPython 开发规则。

  
👉 ​**爽点**​：写代码时 Claude 自动按规范把关，提交前就解决 90% 的规范问题，PR 通过率直接拉满。

### `mpy-ci`：本地一键跑全量 CI，再也不用等 GitHub 排队

**谁懂啊！**MicroPython 支持 18 个硬件端口、8 项代码质量检查，每次改完代码推上仓库，等 CI 跑半小时才发现拼写错误，反复提交真的磨人…

`mpy-ci` 直接把完整的 MicroPython CI 流水线搬到了本地，用 Docker 一键运行，覆盖所有端口构建、测试和代码检查，不用搭任何复杂环境。

👉 **怎么用？直接跟 Claude 说：**

*   「帮我跑一下这个改动的完整 CI 测试」
*   「构建一下所有端口的固件」
*   「运行所有代码质量检查」

它会自动拉取 Docker 镜像，跑完直接给你反馈结果，哪里报错、哪里不通过一目了然。

👉 ​**爽点**​：本地提前发现所有问题，不用反复推仓库等 CI，开发效率直接翻倍。

### PR 管理双雄：从「分拣」到「维护」，全自动搞定 PR 积压

做开源开发，谁没一堆挂在那儿的 PR？新的旧的、要反馈的、要重基的，翻 GitHub 翻到眼瞎都分不清优先级…

这个市场直接给你做了一套 PR 全流程工具：

#### `mpy-pr-triage`：你的 PR 待办「智能分拣员」

一键列出你所有的开放 PR，按「需要反馈 / 最新 / 最旧」排序，还能实时同步分支和工作区状态，直接告诉你哪些 PR 需要优先处理。👉 用法：打 `/mpy-pr-triage` 或者问 Claude「我有哪些 PR 需要处理？」，它直接给你整理好带优先级的列表。

#### `mpy-pr-maintenance`：PR 维护「全自动流水线」

光分拣还不够，维护 PR 才是真的折磨：手动重基、解决冲突、跑本地 CI、再强制推送，每个步骤都要敲一堆命令…

这个插件给 PR 维护做了一套系统化工作流：从旧到新处理你的 PR，自动完成状态侦查、用户讨论、工作区重基修复、本地 CI 验证，最后直接帮你推送到远程，还能批量处理没人反馈的 PR。  

👉 ​**爽点**​：PR 维护从「手动流水线」变成「一键自动化」，你只需要确认关键步骤，剩下的交给 Claude。

### `mpremote`：嵌入式调试的「懒人神器」

用 MicroPython 开发，调试设备总绕不开 `mpremote`：连设备、传文件、跑脚本、看串口输出，每次都要记一堆命令，会话断了还要重连…

这个插件把 `mpremote` 的核心功能做成了三个对话式技能：

*   设备连接与管理：自动识别设备、建立连接
*   文件传输：直接说「把这个文件传到设备的 `/lib` 目录」就行
*   持久化 PTY 会话：开一个 asyncio 安全的持久会话，不用每次调试都重连

👉 用法：直接跟 Claude 说「帮我连到这个设备上跑一下 `test.py`」「监控一下设备的串口输出」，它自动帮你处理所有命令，调试终于不用手敲了！

### `mpy-reviewer`：代码 Review 的「上游级导师」

提交 MicroPython 代码，最怕的就是被上游 Reviewer 打回，不知道哪里不符合要求？

`mpy-reviewer` 是一个基于上游 Review 评论训练的 RAG 助手，还带 MCP 服务器可以查 Review 数据库，能给你专业的、符合上游标准的代码修改反馈。

👉 用法：问 Claude「帮我 Review 一下这段 MicroPython 代码，看看有什么问题」，它直接给你针对性的 Review 意见，提前避坑。

三、两种安装方式，新手 / 开发者都能搞定
---------------------

不用怕复杂，安装方式超简单，分两种情况：

### 👉 普通用户：一键安装，重启就生效

直接在 Claude Code 里敲几条命令：

    claude plugin marketplace add https://github.com/andrewleech/claude-mpy-marketplace 
    claude plugin install mpy-rules@mpy-marketplace 
    claude plugin install mpy-reviewer@mpy-marketplace 
    claude plugin install draft-pr@mpy-marketplace 
    claude plugin install mpremote@mpy-marketplace
    

装完重启 Claude Code，所有插件就都能用了，全程不超过 5 分钟。

### 👉 开发者：本地调试，实时更新

想改插件或者本地调试？直接克隆仓库：

    git clone https://github.com/andrewleech/claude-mpy-marketplace.git 
    cd claude-mpy-marketplace 
    # 一键启用所有插件 
    ./install.sh --enable-all 
    # 或者先注册，之后手动启用 
    ./install.sh
    

最爽的是：修改插件文件后，下次会话就会自动生效，不用重装，调试超方便。

四、想自己写插件？规则超简单
--------------

如果你有自己的开发流程，想给 Claude 加自定义技能？这个市场的插件结构超清晰，几步就能搞定：

1.  在 `plugins/` 目录下建你的插件文件夹
2.  放 `.claude-plugin/plugin.json` 配置文件
3.  把技能写在 `skills/<技能名>/SKILL.md` 里，命令写在 `commands/<命令名>.md` 里
4.  注册到 `.claude-plugin/marketplace.json`
5.  跑 `./install.sh --enable-all` 启用就行

相当于给你的开发流程加自定义对话式技能，怎么顺手怎么来。
