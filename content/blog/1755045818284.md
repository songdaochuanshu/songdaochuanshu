【译】Visual Studio 2015 停用：针对旧版本 Visual Studio 的支持提醒
==================================================

　　对 Visual Studio 2015 的支持将于2025年10月14日结束。如果您正在使用较旧版本的 Visual Studio，我们希望在您使用 Visual Studio 时保持您的工作效率和安全性，并强烈建议您升级到 Visual Studio 2022，它将 Copilot 的强大功能引入您的开发工作流程。它还在编辑器中进行了关键的生产力改进，内置对 Git 版本控制的支持以及与 GitHub 的深度集成、人工智能驱动的代码自动完成，以及用于 .NET 8 和 .NET 9 以及 .NET MAUI 的工具。

旧版本支持的时间范围提醒
============

　　如果您仍在使用旧版本的 Visual Studio，以下是关键日期的提醒。

　　1. Visual Studio 2015 的扩展支持将于 2025 年 10 月 14 日结束。

　　2. Visual Studio 2017 版本 15.9 的扩展支持将持续到 2027 年 4 月 13 日。在扩展支持期间，我们仅针对安全问题提供修复。您必须使用版本 15.9 才能获得安全更新和支持。

　　3. Visual Studio 2019 版本 16.11 的扩展支持将持续到 2029 年 4 月 10 日。在扩展支持期间，我们仅针对安全问题提供修复。您必须使用版本 16.11 才能接收安全更新和支持。

　　4. Visual Studio 2022 版本 17.14 在 2027 年 1 月 12 日之前处于主流支持阶段，之后将过渡到扩展支持阶段，直至 2032 年 1 月 13 日。此外，版本 17.10 LTSC 的支持期至 2026 年 1 月 13 日，版本 17.12 的支持期至 2026 年 7 月 14 日。您必须使用这些版本之一才能获得更新和支持。

Visual Studio 2015 停用
=====================

　　2025 年 10 月 14 日起，所有 Visual Studio 2015 版本、相关产品、运行时和组件将停止支持，不再接收安全更新或其他任何更新。这些包括：

　　- Visual Studio 2015 Enterprise, Professional, Community, Build Tools, Team Explorer, Test Professional版，以及 Visual Studio 2015 Shell （集成式和独立式）。

　　- Visual Studio SDK、Remote Tools、Agent、适用于 Visual Studio Team Foundation Server 2015 的 Feedback Client、Deployment Agent、Release Managemen 以及 Azure 工具。

　　- MSVC Tools v140  - Visual Studio 2015。使用 Visual Studio 2017 或更高版本时，请更新任何使用 MSVC v140 的项目，以使用最新的 MSVC 工具集。

　　- 适用于 Visual Studio 2015 的 Visual C++ Redistributable。请将应用程序更新到最新版本的 Visual C++ Redistributable。

感谢您的反馈
======

　　您的反馈有助于我们改进 Visual Studio，使其成为开发人员更强大的工具。我们非常感谢您的贡献，并期待您的持续支持。通过在开发者社区分享您的想法、创意以及遇到的任何问题，您将帮助我们改进并塑造 Visual Studio 的未来。
