在鸡哥14x上安装Linux：Fedora 42 上手体验
=============================

前言
--

Fedora 这系统更新很快，版本号刷得飞起

记得上一次看到讨论时才二十几的版本号，没想到现在已经 42 了

之前我说过鸡哥14x送修回来了

*   [不到半年，我的鸡哥14x也翻车了……](https://blog.deali.cn/p/half-year-mechrevel-unbound14x-crashed)
*   [送修10天，鸡哥14x终于回来了，然而事情并没有这么简单……](https://blog.deali.cn/p/repair-10-days-kunkun-14x-back)

之前为了测试按键特地装了个Linux，结果还没用两天就送修了

现在修好了，得来深度使用试试看 2025 年的 Fedora 系统体验咋样。

* * *

本文就是简单记录一下，不涉及具体的配置，Linux 桌面的体验发展速度超乎我的想象~

关于系统安装配置的，可以参考我去年写的文章: [2024年，我又开始用Linux桌面作为主力系统了](https://blog.deali.cn/p/linux-desktop-main-system-2024)

总览
--

维度

亮点 ✨

不足 🤔

安装 & 驱动

系统安装顺利，驱动开箱即用（Wi-Fi / 蓝牙 / 摄像头都能用）

重启后偶尔遇到 Wi-Fi 不识别的问题，需要重装或排查

桌面环境（GNOME）

界面小清新，启动台好用，触控板体验优于 Windows

多任务界面不如 Windows 整齐；小 bug 偶尔出现

终端 & 工具

自带终端颜值高，功能好用；截图工具好用，支持多种导出格式

终端透明度找不到配置入口

文件管理

默认支持 AVIF / WebP，预览和打开无压力

界面简洁但功能偏基础，缺少高级功能（如双栏）

输入 & 交互

鼠标滚动方向可调，触控板边缘操作手势惊喜

Wayland 下仍感觉卡顿，X11 又不够顺滑

稳定性

日常应用大体流畅，驱动兼容度好

Wayland 生态“半死不活”；GNOME 名称错乱小 bug

生态 & 未来

Fedora 更新快，前沿体验，GNOME 有改进

不确定性大（更新频繁，可能引入新 bug）

小结
--

鸡哥14x使用 Linux 的体验还不错

驱动什么的都是开箱即用，除了人脸识别这个我没去折腾，其他都没啥问题

不过 Wayland 不知道为啥搞了这么多年还是半死不活

使用 XWindow 总觉得有点不得劲，相比起 Windows 来说有点卡卡的感觉？

网友建议我试试 KDE，我正在寻找一个合适的发行版（总不会最终还是入坑 Arch 了吧？）

另外 GNOME 虽然好看，但小 bug 也是不少的，比如有时候开启动台，里面 App 的名称显示错乱，重新打开就又好了，不知道是 GNOME 的 bug 还是 Wayland 的 bug 😂

对了，隔了一段时间没用，这次再次进入 Fedora 系统发现网络坏了，Wi-Fi 识别不到，插网线也不认，不知道是哪里出问题了。我切换了内核版本也没用……不过我打算重装了，不管那么多了。

OK，就这样吧，我要抓紧时间去试用 [Pop!\_OS](https://system76.com/pop/) 了 😄

番外：关于 Pop!\_OS
--------------

这是一个很有特色和潜力的发行版，为什么想要试试这个，起因是我由想起了 Unity 桌面，之前我在 Ubuntu 16.04 停留了很久一直不舍得换掉这老古董，就是因为 Unity 桌面，后面 18.04 开始就砍掉了，Ubuntu 母公司也放弃了这个项目，真是可惜，我觉得 Unity 是 Linux 生态体验最好的桌面。

我去论坛冲浪的时候，看到很多网友也同样喜欢 Unity，刷着刷着就看到了 Pop!\_OS 了，好奇怪的名字，网友评价这是能扛起 Linux 桌面大旗的发行版，基于 Ubuntu ，可以用熟悉的 apt 包管理器，提供了自研的 COSMIC 桌面作为 GNOME 和 KDE 之外的选择，还有自带的平铺窗口管理，这些都值得尝试。

而且 Pop!\_OS 背后是硬件厂商 System76，收入能保障系统的开发，不像 Canonical 一样穷得要死把 Unity 和 mir 都砍了🤣

话不多说，赶紧发完文章来重装了

番外：关于截图格式
---------

本文的图片都由 GNOME 自带的截图工具完成

默认截图的格式是 png

总共 11 张图片，我使用 magick 测试了一下不同格式的大小，仅供参考

格式

大小

png

6.42 MB

jpeg

3.14 MB

webp

777 KB

avif

455 KB

使用 ImageMagick 转换，还挺方便的

这里我也附上脚本

Shell 版本

    for i in *.png; do
        magick "$i" "output/${i%.png}.jpg"
    done
    

PowerShell 版本

    foreach ($i in Get-ChildItem *.png) {
        magick $i.FullName ("output\" + $i.BaseName + ".jpg")
    }
    

avif 版本的最小，但转换速度也最慢

webp 是这里面最均衡的，性价比最高，兼容性也好。我的博客也是用的 webp 这种方案。
