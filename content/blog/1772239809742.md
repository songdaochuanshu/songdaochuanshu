### Perplexity Computer：编排式 Agent 平台

Perplexity 推出了名为 **Computer** 的端到端系统，定位为"编排优先"(orchestration-first)的 Agent 产品：

*   **产品定位**：能够"研究、设计、编码、部署和管理"项目，通过编排文件、工具、记忆和模型实现一体化界面
*   **定价策略**：Web 端仅向 Max 订阅用户开放，后续开放 Pro/Enterprise；采用使用量计费模式
*   **架构创新**：多个并行、异步的子 Agent 协调工作，Coordinator 模型分配任务给专业模型（研究/编码/媒体）
*   **核心理念**："一切皆计算机"——小团队通过大量使用编码 Agent 和自动化评估/调试循环构建的平台

**为什么对工程师重要**：Computer 代表了系统级 Agent UX 的具体推进，包括多模型路由、隔离/沙箱、持久记忆和成本控制——即把"Agentic 工作"视为**分布式工作流**而非单一聊天会话。

### Karpathy 的"相变"论断

Andrej Karpathy 认为 **编码 Agent 自 12 月以来跨越了质变阈值**——从脆弱的演示变成了连贯且坚持的长周期任务完成。

他展示了一个详细示例：委托端到端本地部署（SSH 密钥 → vLLM → 模型下载/基准 → 服务器端点 → UI → systemd → 报告），只需极少的干预。

这一观点与更广泛的"软件正在改变"共识一致，来自开发工具构建者和用户（如 Cursor、Snowmaker 等）。
