MCP 实践系列：EdgeOne 在线部署HTML页面
===========================

今天，我们将专门讲解一下 EdgeOne 在线部署 HTML 页面时所提供的 MCP 功能。这个功能对于个人用户来说非常实用，尤其适合一些小型应用场景，比如开发一个简单的小游戏，或者搭建一个小型网站，展示一些基本的站点信息等。通过 EdgeOne 的 MCP 功能，我们可以将静态页面直接部署到服务器上，而无需承担额外的费用。更为重要的是，它免去了我们为实现这样一个简单需求而购买低配服务器的成本。

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172511730-792855453.png)

我们仍然以腾讯云的编程助手 CodeBuddy 作为 MCP 客户端，通过它进行直接调用演示。在这个演示过程中，我们将展示如何利用 CodeBuddy 轻松地与 EdgeOne 的 MCP 功能进行对接，并实现在线部署 HTML 页面。

前期准备
====

如果你本地尚未在IDE编辑器中下载并安装CodeBuddy助手，请先按照以下步骤进行下载和安装。如图所示，完成相关操作后，你就可以开始使用CodeBuddy助手了：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172515541-195913198.png)

接下来，我们将按照如下步骤，在MCP市场中安装并配置EdgeOne的MCP服务，具体过程如下图所示：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172520494-474777128.png)

请注意，直接运行这个服务之前，首先要确保你的本地环境已经配置好Node.js。特别是需要安装Node.js版本18及以上。如果你的本地没有安装Node.js，建议你先安装一个版本管理工具——NVM（Node Version Manager），这个工具特别适用于Windows操作系统。通过NVM，你可以轻松管理多个Node版本，并根据需要切换不同版本的Node.js。比如，在我的本地环境中，就安装了两个不同版本的Node.js，如图所示：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172544898-2071274769.png)

如果在本地安装时没有选择安装18版本的 Node.js 环境，虽然仍然可以完成安装过程，但可能会导致无法成功运行 MCP 服务。在这种情况下，您只需刷新 MCP 服务即可解决问题，操作步骤如图所示：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172538095-2018914082.png)

测试
==

一切准备工作已经顺利完成，我们无需申请额外的密钥信息。接下来，我们将在此直接部署我本地上次通过 AI 生成的一个简单的介绍 MCP 的页面。部署完成后，如图所示：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172556434-907478276.png)

在Craft标签下，我们将直接提供完整的HTML页面。在此过程中，调用MCP服务时可能会涉及到一定的延迟，因此需要稍作等待。经过短暂的处理时间后，系统会返回一个对外可访问的路径，具体示例如下：

> [https://mcp.edgeone.site/share/lbml8fes8hJYEJ78h0RlA](https://mcp.edgeone.site/share/lbml8fes8hJYEJ78h0RlA)

我们直接访问下，看下效果如何：

![image](https://img2024.cnblogs.com/blog/1423484/202505/1423484-20250505172601964-118366159.png)

请注意，在此过程中，系统默认会自动添加一个类似水印的小标签，虽然该标签会被显示在页面中，但用户可以手动将其删除。需要特别说明的是，目前系统不支持配置自动删除功能。

小结
==

EdgeOne 的 MCP 功能为开发者提供了一个高效、便捷的在线部署解决方案，特别适合个人用户和小型项目。通过 CodeBuddy 客户端，用户可以轻松将静态 HTML 页面部署到服务器，避免了购买服务器的额外成本。虽然系统会在页面上添加水印标签，但这一小小的限制并不妨碍其强大的实用性。未来，随着功能的不断完善，EdgeOne 有望为更多开发者带来更大的便利。
