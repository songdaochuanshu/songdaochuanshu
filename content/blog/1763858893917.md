LaTeX学习笔记：文档排版基础
----------------

LaTeX 学习笔记：文档排版基础
=================

在撰写学术论文、技术报告或专业著作时，排版的质量往往决定了作品给人的第一印象。对于计算机科学、工程技术、数理化等领域而言，文档中不仅包含大量数学公式，还常常涉及图表、参考文献等复杂结构。正如我们在《\[\[LaTeX学习笔记：开场白与索引\]\]》一文中所提到的那样，计算机科学家们正是为了应对这些复杂的排版需求而开发了 \\(\\LaTeX\\)。

在本篇笔记中，我们将重点介绍 \\(\\LaTeX\\) 在专业排版中的基础技巧，帮助提升学术文档的可读性与专业气质。下面，就让我们先从文档的排版结构开始吧。

定义排版结构
------

在 \\(\\LaTeX\\) 中，文档的整体排版结构是通过一系列**结构性命令**（structural commands）来定义的。这些命令用于给文档指定排版类型，加载功能扩展包，设置页面版式，设置标题、作者等元信息，并生成文档的开篇部分。最常用的命令包括：

*   `\documentclass`：用于指定文档所要采用的排版类型；
*   `\usepackage`：用于通过加载宏包的方式来获得扩展功能；
*   `\geometry`：用于设置文档的页面尺寸与页边距等页面版式信息
*   `\title`、`\author`、`\date`：用于定义文档的标题、作者等元信息；

接下来，我将会按照功能逐一介绍这些命令的使用方法。

### 指定排版类型

在使用 \\(\\LaTeX\\) 语言定义一个文档的排版结构时，我们需要先在其源文件的第一行中使用`\documentclass`命令指定要采用的排版类型。换而言之，我们需要首先指定该文档的排版规格属于短篇文章、学术报告、信件、书籍，还是他类型的文献。在程序员的日常工作中，较为常用排版类型主要可分为以下几种：

1.  **标准文档类**，由 \\(\\LaTeX\\) 核心包提供，主要包括以下四个类：
    
    *   `article`：用于排版短篇文章，包括期刊投稿、技术文档等；
    *   `report`：用于排版学术报告，包括毕业论文、技术报告等；
    *   `book`：用于排版书籍类文档，例如专著、教材等；
    *   `letter`：用于排版信件类文档，包括个人书信、商务信函等；
2.  **扩展文档类**：由`extsizes`宏包提供，提供有`extarticle`、`extreport`、`extbook`三个类，它们分别是上述前三个标准文档类的扩展版，主要区别是支持更大的字号选项（如 `14pt`、`17pt`、`20pt`）
    

3.**中文文档类**：由`ctex`宏包提供，提供有`ctexart`、`ctexrep`、`ctexbook`、`ctexletter`四个类，它们分别是上述四个标准文档类的中文增强版本，可自动配置中文字体、字号、段落间距与标题格式，更适合中文排版。值得一提的是：在新版 CTeX 宏包中（自 2.0 以后），`ctexletter`类尽管仍然存在，但默认情况下不会随发行版安装，读者如有需要可通过 texdoc ctex 查阅详细信息。

下面，让我们来具体演示一下`\documentclass`命令的使用。假设，你现在想要让目标文档按照`ctexart`类型来排版，那么就需要在其 \\(\\LaTeX\\) 源文件的开头添加以下命令：

    \documentclass{ctexart}
    

当然，该命令除了指定文档的排版类型之外，还可以通过一些可选参数来指定文档的某些排版细节。例如像下面这样：

    \documentclass[12pt,oneside]{ctexart}
    

在上述命令的中括号里，参数`12pt`用于将文档的字体大小设置为 12 磅（point），而参数`oneside`则用于指定该文档将采用单面打印的方式输出。总体来说，`\documentclass`命令的可选参数主要可分为以下几大类：

1.  **字体与字号参数**：用于设置文档的字体大小，除了之前提到的 `12pt` 外，还包括如下常见选项：
    
    *   `10pt`（默认值）：适用于一般文档；
    *   `11pt`：适用于需要稍大字体的论文或报告；
    *   `14pt`, `17pt`, `20pt`：部分扩展类支持更大的字号；
    
    需要再次强调的是：`14pt`、`17pt`、`20pt`等较大字号仅在`extarticle`等扩展类中可用。另外，对于中文类文档（如 `ctexart`），也可通过 `zihao` 参数控制字号，例如：
    
        \documentclass[zihao=-4]{ctexart} % 设置中文字号
        
    
    这会根据中文字号标准（初号、四号等）调整排版比例，更符合中文出版规范。
    
2.  **页面尺寸参数**：用于设置文档的页面尺寸，常见选项主要包括：
    
    *   `a4paper`：适用于 A4 纸；
    *   `letterpaper`：适用于 US Letter 纸；
    *   `a5paper`：适用于 A5 纸；
    *   `b5paper`：适用于 B5 纸；
    *   `executivepaper`：适用于 Executive 纸；
    *   `legalpaper`：适用于 Legal 纸；
3.  **页面布局参数**：用于设置文档的页面布局，常见选项主要包括：
    
    *   `titlepage/notitlepage`：控制是否生成独立标题页（`report`和`book`这两个类默认采用`titlepage`，`article`类默认则不是）；
    *   `openright/openany`：控制章节起始页是否必须在奇数页（书籍排版常用）；
    *   `twocolumn/onecolumn`：双栏或单栏排版；
    *   `draft/final`：在草稿模式下，LaTeX 会标出溢出行、图片留空；
    *   `fleqn/leqno`：左对齐公式或在左侧编号；
4.  **打印与装订参数**：用于设置文档的打印与装订方式，常见选项主要包括：
    
    *   `oneside/twoside`：控制单双面 排版（`book`类默认采用`twoside`）；
    *   `landscape`：横向排版（与 PDF 输出相配合时尤其有用）。

当然，除了上面列出的排版类型及其相关参数之外，一些特定的学术期刊或机构可能会要求我们按照特排版类型来排版，例如`IEEEtran`、`acm_proc_article-sp`等。虽然在通常情况下，这些来自第三方的排版类型也只需使用`\documentclass`命令来指定即可，但在某些特定情况下，如果想让这些排版类型充分发挥作用，还需通过`\usepackage`命令来加载它们所通过的功能扩展包，以扩展 \\(\\LaTeX\\) 的排版功能，下面就让我们继续来看这个命令的使用方法吧。

### 加载功能扩展包

从程序员的角度，我们可以将 \\(\\LaTeX\\) 排版系统中的`\usepackage`命令理解成 Python 中的`import`语句，C/C++ 语言中的`#include`语句，它们的作用是在当前源文件中引入 \\(\\LaTeX\\) 自带或者第三方提供的扩展包，以便获得特定的功能。在 \\(\\LaTeX\\) 语言中，这些扩展包都是以“宏包（package）”的形式提供的，例如`amsmath`、`graphicx`、`hyperref`等。具体来说，就是我们需要在使用`\documentclass`命令指定了当前文档所要采用的排版类型之后，就紧接着使用`\usepackage`命令来加载我们需要的宏包，以便获得这些宏包提供的扩展功能。

例如，如果我们基于某种灵活性方面的考虑，既想使用标准类`article`排版，又想在文档中使用中文，就可以通过`\usepackage`命令加载`ctex`这个宏包来实现，像下面这样：

    \documentclass{article}
    \usepackage{ctex}
    % 其他命令 ...
    

除此之外，为了达到专业排版效果，我们通常还会推荐加载以下宏包（具体请根据实际需求来进行选择）：

    % 基础
    \usepackage{ctex}        % 中文支持
    \usepackage{geometry}    % 页面设置
    
    % 数学相关
    \usepackage{amsmath,amssymb}
    
    % 图形相关
    \usepackage{graphicx,float,caption,subcaption}
    
    % 表格相关
    \usepackage{booktabs}
    
    % 超链接与引用相关
    \usepackage{hyperref,cleveref}
    
    % 文献管理相关
    \usepackage{biblatex}
    
    % 微排版相关
    \usepackage{microtype}
    

当然，在加载宏包时，需要注意遵循“少而精”原则：只加载必要宏包即可，以避免发生冲突或冗余。例如，在上述推荐中，如果我们事前已经加载了`ctex`宏包，那么`inputenc`、`fontenc`、`lmodern`这几个宏包就无需再加载了。因为使用`ctex`宏包时，我们采用的排版引擎大概率是 XeLaTeX 或 LuaLaTeX，它们已经内置了 UTF-8 编码、T1 字体编码以及 Latin Modern 字体，无需再额外加载。

另外，加载宏包也需要特别注意一下加载的顺序，因为某些宏包之间存在着特定的依赖关系，对加载的顺序是敏感的，例如，我们通常建议先加载数学宏包（`amsmath`、`amssymb`），再加载`hyperref`；而`cleveref`则必须在`hyperref`之后加载。总而言之，读者最好在具体使用之前查阅一下这些宏包的文档，以了解它们之间的依赖关系。

### 设置页面版式

之前在介绍`\documentclass`命令时，我们曾提到过一些与页面版式相关的可选参数，例如`a4paper`、`twoside`、`oneside`等。这些参数属于文档类中的全局配置接口，只能对排版进行一些**粗粒度的默认设置**，例如采用的纸张类型、单/双面打印模式等，但并不能精确控制页面边距或版心尺寸。

如果希望文档在阅读体验和排版细节上达到专业水准，就需要对页面布局进行更精细的控制。通常的做法是先通过`\usepackage`命令加载名为`geometry`的宏包，然后再使用`\geometry`命令设置页面的具体版式。例如像下面这样

    \documentclass{article}
    \usepackage{geometry} % 加载 geometry 宏包
    
    \geometry{
        a4paper,       % 设置纸张尺寸
        left=3cm,      % 左页边距
        right=3cm,     % 右页边距
        top=3cm,       % 上页边距
        bottom=3cm     % 下页边距
    }
    

与`\documentclass`命令中的可选参数相比，`\geometry`命令提供了更细粒度的页面布局控制，内容包括页边距、版心大小、装订线、页眉页脚高度等。因此，它广泛应用于技术文档、学术论文与书籍排版中。下面，让我们来看一个用于给专业书籍排版的示例：

    \documentclass[12pt,twoside]{book} % 双面打印常用于书籍排版
    \usepackage{geometry}
    
    \geometry{
        paper=a5paper,          % 书籍常用 A5 尺寸
        inner=2.8cm,            % 订口（内侧边距）
        outer=2.2cm,            % 外侧边距
        top=2.5cm,              % 上边距
        bottom=2.5cm,           % 下边距
        headheight=14pt,        % 页眉高度
        headsep=0.8cm,          % 页眉到正文距离
        footskip=1.2cm          % 页脚到正文距离
    }
    

在上面这个示例中，我们通过`\geometry`命令对文档进行了精确的页面布局设置，其中：

*   参数`twoside`用于将书籍设置为“奇偶页镜像布局”，即将内侧边距设置得更大一些，以便装订。
*   参数`inner/outer`是专为双页面打印的选项，用于取代参数`left/right`相对死板的页边距设置。
*   `headsep`、`footskip`等参数的作用是用于细调页眉页脚与正文间的距离。

通过这种方式，我们可以在维持文档类整体结构的基础上，对页面版式进行专业级的精细化定义。当然，`\geometry`命令可设置的参数远不止这些，如果读者想了解更多该命令的参数，可以查阅该`geometry`宏包的官方文档。

另外在 \\(\\LaTeX\\) 语言中，关于版式设置的命令也并不只有`\geometry`。还包括用于设置行距、页眉页脚、分栏等多方面的命令或宏包。下面让我们来做个简单的了解：

1.  **`\linespread`命令**：主要利用行间距调整正文可读性，常用于论文或技术报告。例如`\linespread{1.2}`。通常情况下，`1.0`属于正常行间距，大于`1.0`则是选择拉开行距，小于`1.0`则属于紧凑行距。当然，相同的功能也可以通过加载 `setspace` 宏包来实现，例如：
    
        \usepackage{setspace}
        \singlespacing     % 单倍行距
        \onehalfspacing    % 1.5 倍行距
        \doublespacing     % 双倍行距
        
    
2.  **`\pagestyle`命令**：用于设置页眉页脚的样式，它的参数可以是`plain`、`empty`、`headings`、`myheadings`等。其中，`plain`是默认的页眉页脚样式（只显示页码）；`empty`则不显示页眉页脚；`headings`则会在页眉中显示章节标题和页码；`myheadings`则允许用户自定义页眉内容。同样的，`\pagestyle`命令的功能也可以通过加载`fancyhdr`宏包来得到扩展，例如：
    
        \usepackage{fancyhdr}
        \pagestyle{fancy}
        \fancyhf{} % 清空页眉页脚
        \fancyhead[L]{左页眉}
        \fancyhead[C]{中页眉}
        \fancyhead[R]{右页眉}
        \fancyfoot[L]{左页脚}
        \fancyfoot[C]{中页脚}
        \fancyfoot[R]{右页脚}
        
    
    在上述示例中：
    
    *   `\fancyhf{}`命令用于清空页眉页脚；
    *   `\fancyhead`命令用于设置页眉内容；
    *   `\fancyfoot`命令用于设置页脚内容。
    *   `[L]`、`[C]`、`[R]`分别表示左、中、右三个位置。
    *   `fancy`参数表示采用`fancyhdr`宏包提供的页眉页脚样式。
3.  **`\twocolumn`、`\onecolumn`**命令：用于控制正文是否双栏排版。例如`\twocolumn`是双栏排版，`\onecolumn`则是单栏排版。
    
4.  **`\topmargin`、`\oddsidemargin`等命令**：它们是传统 LaTeX 提供的命令，可直接控制页边距，例如：
    
        \setlength{\topmargin}{-0.5in}
        \setlength{\oddsidemargin}{0.2in}
        \setlength{\textwidth}{6in}
        \setlength{\textheight}{9in}
        
    
    **请注意**：上述命令使用起来不如`\geometry`命令直观，很容易出错，现代文档推荐使用`geometry`宏包提供的功能。
    

除了上面列出的这些命令和相关宏包之外，与版式相关的宏包还包括以下这些，读者如有需要可以查阅相关文档：

*   `titlesec`：用于设置章节标题样式；
*   `caption`：用于设置图表标题格式；
*   `parskip`：用于设置段落间的垂直间距；
*   `microtype`：用于字间距微调，提高排版质量；
*   `layout`：用于设置可视化页面布局;
*   `typearea`：用于设置页面版式，与`geometry`类似，但功能更强大。

### 设置文档元信息

在 \\(\\LaTeX\\) 语言中，文档可设置的元信息取决于其采用的排版类型。例如，就之前提到的`article`、`report`、`book`等文档类而言，我们分别可通过`\title`、`\author`、`\date`这三个命令来设置文档的标题、作者与发表日期，这些信息通常会显示在文档的标题页中。例如，如果我们想对张三撰写的一篇题为“LaTeX 快速入门”的文档进行排版，就可以像这样设置它的元信息：

    \documentclass[oneside, 12pt, a4paper]{article}
    \usepackage{ctex} % 支持中文
    
    % 设置文档元信息
    \title{\LaTeX 快速入门} % 设置文档标题
    \author{张三}           % 设置文档作者，如果有多个作者，可使用“\and”分隔，
                            % 例如：“张三\and 李四”
    \date{\today}           % 设置文档日期，如果不想设置，可以写 \date{}
    
    % 开始组织文档内容
    \begin{document}
        \maketitle          % 生成标题页
    \end{document}

*   `beamer`文档类提供了用于设置副标题（`\subtitle`命令）、机构（`\institute`命令）、图标（`\logo`命令）等元信息的扩展接口；
*   `IEEEtran`文档类提供了用于设置关键字（`IEEEkeywords`环境）、摘要（`abstract`环境）等元信息的扩展接口；
*   `memoir`文档类提供了`\publishers`、`\affiliation`、`\subject`等更丰富的标题页定制命令，分别用于设置出版商、机构、主题等元信息；

换而言之，如果要设置相对复杂的元信息，需要根据对应文档所采用的排版类型及其相关的文档说明来设置，并不存在通用的解决方案。另外，需要额外说明的一点是：**标题页上显示的元信息**（例如 `\title`命令输出的内容）与**PDF 文件本身的元信息** 并不是同一套东西。后者常见于 PDF 阅读器的“文件属性”界面，需要使用`hyperref`宏包来设置，例如：

    \usepackage{hyperref}
    \hypersetup{
        pdftitle={LaTeX 快速入门},
        pdfauthor={张三},
        pdfsubject={LaTeX 教程},
        pdfkeywords={LaTeX, 排版, 入门}
    }
    

这些属性不会根据`\title`或`\author`命令来自动推断，因此通常需要手动指定。总体来说，**\\(\\LaTeX\\) 的元信息系统是“按需扩展”的**：基础类只提供最小集合，而不同的文档类和宏包根据场景添加自己的元数据接口。这也是 \\(\\LaTeX\\) 在学术论文、演示文稿、技术报告等场景中具有高度可扩展性的原因。

组织文档内容
------

一般来说，如果我们需要用到 \\(\\LaTeX\\) 这种复杂的系统来排版一份文档，那么该文档通常会是一篇有相当规模和专业性的作品，例如学术论文、技术报告、出版书籍等。这些作品中往往会包含封标题页、目录结构、章节层级、正文段落、插图、表格、参考文献等元素。这些元素通常都有着来自出版社、学术机构的，既定且严格的格式规范。接下来，我们将以一篇文档的实际编写过程为例来逐一介绍这些元素在 \\(\\LaTeX\\) 中的基本排版方式。

### 章节层级设计

在正式开始撰写一篇文章之前，我们通常需要先拟定相应的写作大纲。在 \\(\\LaTeX\\) 的排版系统中，这项工作可以通过设计文档的章节层级来完成。当然，这需要根据我们在定义文档结构时所选择的排版类型来进行，因为它们使用到的命令是略有差异的。譬如在之前介绍的标准文档类中，`article`类文档的章节层级通常如下：

*   **一级标题**：由`\section`命令定义。
*   **二级标题**：由`\subsection`命令定义。
*   **三级标题**：由`\subsubsection`命令定义。
*   **四级标题（可选）**：由`\paragraph`命令定义。

而在`report`和`book`类的文档中，排版系统在上述命令之外还提供了级别更高的`\chapter`命令，和级别更低的`\subparagraph`命令，因此它们的章节层级一般包括：

*   **一级标题**：由`\chapter`命令定义。
*   **二级标题**：由`\section`命令定义。
*   **三级标题**：由`\subsection`命令定义。
*   **四级标题**：由`\subsubsection`命令定义。
*   **五级标题（可选）**：由`\paragraph`命令定义。
*   **六级标题（可选）**：由`\subparagraph`命令定义。

**请注意**：在上述标准文档类中，`\paragraph`和`\subparagraph`这两个命令所显示的文本属于 run-in heading（即标题后紧跟着的正文），在默认情况下是不会自动换行的，所以它们更接近“带标题的段落”。并且，它需手动调整目录深度后才会被纳入目录。关于目录的生成，我们稍后会具体介绍。

下面，让我们以“\\(\\LaTeX\\) 排版示例”为题，来演示一下如何基于`article`类文档来设计章节层级，其具体代码如下：

    \documentclass[oneside, 12pt, a4paper]{article}
    
    \usepackage{ctex}     % 用于提供中文支持
    \usepackage{geometry} % 用于设置页面版式
    
    % 设置页面边距
    \geometry{a4paper, left=3cm, right=3cm, top=3cm, bottom=3cm}
    % 设置文档元信息
    \title{\LaTeX 排版示例}
    \author{凌杰}
    \date{\today}
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            \subsection{正文段落}
            \subsection{引言文本}
            \subsection{列表项目}
            \subsection{注释说明}
        \section{图表元素示例}
            \subsection{插图元素}
            \subsection{表格元素}
        \section{专业内容示例}
            \subsection{定理证明}
            \subsection{数学公式}
            \subsection{算法描述}
            \subsection{代码高亮}
    \end{document}
    

而编译成 PDF 的输出效果如图2所示：

在上述示例中，除了之前介绍过的文档结构定义部分之外（可理解为文档的全局设置区），我们将用于组织文档内容的命令都放在了`\begin{document} ... \end{document}`这个环境定义内。在 \\(\\LaTeX\\) 语言中，文档内容的定义必须放在这个名为`document`的环境内，否则编译器会报错。这是因为`document`环境是 \\(\\LaTeX\\) 语言中用于定义文档内容的唯一环境，它告诉编译器从哪里开始处理文档内容。

而在`document`环境内，我们将文章分成了“正文样式示例”、“图表元素示例”和“专业内容示例”三个部分，每个部分都由一个一级标题（`\section`命令）来定义。每个一级标题下又分别定义了若干个二级标题（`\subsection`命令），它们共同组成了这篇文章的章节层级。

另外，如果我们在这里需要调整各章节的标题样式，可通过加载`titlesec`宏包。例如，通过下面这段代码，我们可以将使用`\section`命令定义的一级标题设置为大号粗体：

    % 如果是中文文档，建议先加载 ctex 宏包
    \usepackage{titlesec}
    \titleformat{\section}{\Large\bfseries}{\thesection}{1em}{}
    % 在 \begin{document} ... \end{document} 之前添加上述代码
    

**请注意**，我们在对专业文档进行排版时，务必要注意整体排版风格的一致性，原则上不宜随意混用多种风格，或进行过多的局部细节调整。尤其`titlesec`宏包与`ctex`、`hyperref`这些宏包在搭配处理中文标题时，可能还会出现一些意想不到的问题，更需谨慎使用。如果遇到非用不可的情况，建议先查阅相关文档，先搞清楚这些宏包的加载顺序，以及它们之间的相互影响，再进行尝试。

### 正文内容排版

在确定了写作大纲之后，接下来就可以在这些既定的章节标题下面按部就班地填写正文内容了。下面，就让我们继续以“\\(\\LaTeX\\) 排版示例”这篇文章为例，来介绍如何使用 \\(\\LaTeX\\) 来排版它的正文内容。

    }
    % 开始组织文档内容

#### 引言文本

在写作过程中，我们经常会需要以引言的形式借用第三方的言论来增强文章的说服力。在 \\(\\LaTeX\\) 语言中，我们可以通过定义一个名为`quote`环境来设置引言形式的文本。这种形式的文本会自动呈现出相对于其他文本进行左右缩进的排版效果。例如，如果我们想在上述文章中的“引言文本”这个二级标题下面添加一段引言，并以右对齐的方式声明引言的出处，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            \subsection{正文段落}
            %... 省略之前的正文段落内容
            \subsection{引言文本}
                \noindent 下面将显示的是一段引言文本的示例：
                \begin{quote}
                    \textit{“\LaTeX 是一种排版系统，它使用TeX排版引擎，并遵循一套文档类和宏包的规范，以实现文档的自动化排版。”}
            
                    {\raggedleft -- 高德纳（Donald E. Knuth） \par}
                \end{quote}
        %... 省略后续内容
    \end{document}
    

在上述示例中，我们首先使用了`\begin{quote} ... \end{quote}`这个环境定义设置了一段引言文本。在`quote`环境内部，我们可以使用普通的文本命令（如`\textit{}`、`\underline{}`等）来设置这段文本的样式。接下来，我们用`\raggedleft`命令将引言的来源设置成了右对齐的文本，与引言正文保持一定的间距（这里使用`\hfill`命令也可以实现相同的效果）。最后，我们使用`\par`命令结束了`\raggedleft`命令的使用效果，以防它的排版效果意外影响到其他文本。

当然，`quote`环境通常只用于设置单一段落的引言文本，如果需要设置包含多个自然段的较长文本，`quotation`环境可能是更合适的选择。例如，如果我们想继续在之前的引言后面再引用来自维基百科的两段文本，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            \subsection{正文段落}
            % 省略之前的正文段落内容
            \subsection{引言文本}
            % 省略之前的引言文本内容
    
            \noindent 下面是一个较长的引用文本示例：
            \begin{quotation}
                \textit{
                    “Leslie Lamport 在 1980 年代初期开发了 LaTeX 系统，以简化 TeX 的使用。LaTeX 提供了一套宏和命令，使用户能够专注于文档内容，而无需过多关注排版细节。LaTeX 迅速成为学术界和技术文档撰写的标准工具，尤其在数学、计算机科学等领域得到了广泛应用。
                    
                    LaTeX 的设计理念强调结构化文档和内容优先，这使得它在处理复杂文档时表现出色。通过使用各种宏包，用户可以轻松地扩展 LaTeX 的功能，以满足不同的排版需求。今天，LaTeX 已经发展成为一个庞大的生态系统，拥有丰富的资源和活跃的社区支持。”
                }
                {\raggedleft -- 维基百科（Wikipedia） \par}
            \end{quotation}
        %... 省略后续内容
    \end{document}

#### 列表元素

在写作过程中，我们有时会需要以列表的形式展示一些有序或无序的选项、步骤或要点内容。在 \\(\\LaTeX\\) 语言中，我们可以分别通过`itemize`、`enumerate`、`description`这三个环境来设置相应的列表元素。例如，如果我们想在上述文章中的“列表元素”这个二级标题下面添加一个有序列表、一个无序列表以及一个描述列表，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            % 省略之前的章节
            \subsection{列表项目}
                \noindent 下面是一个有序列表的示例：
                \begin{enumerate}
                    \item 第一个选项
                    \item 第二个选项
                    \item 第三个选项
                \end{enumerate}
    
                \noindent 下面是一个无序列表的示例：
                \begin{itemize}
                    \item 第一个选项
                    \item 第二个选项
                    \item 第三个选项
                \end{itemize}
    
                \noindent 下面是一个描述列表的示例：
                \begin{description}
                    \item[第一个选项] 这是第一个选项的描述内容
                    \item[第二个选项] 这是第二个选项的描述内容
                    \item[第三个选项] 这是第三个选项的描述内容
                \end{description}
        %... 省略后续内容
    \end{document}
    

在上述示例中，我们分别使用了`\begin{enumerate} ... \end{enumerate}`、`\begin{itemize} ... \end{itemize}`、`\begin{description} ... \end{description}`这三个环境来设置有序列表、无序列表以及描述列表。在列表内部，我们可以使用`\item`命令来定义每个列表项的内容。需要注意的是，在`description`环境中，每个列表项的内容前面会自动加上一个描述标签，我们可以通过`\item[标签名]`命令来设置这个标签的名称。

如果读者还想设置更为复杂的多层次列表，就需要通过嵌套定义另一个名为`enumerate`的环境来实现了。该环境在默认情况下最多允许我们嵌套定义四层列表，\\(\\LaTeX\\) 会自动为每个列表项添加相应的编号。例如，如果我们想在上述文章中接着上面的描述列表添加一个三层嵌套的多层次列表，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            % 省略之前的章节
            \subsection{列表项目}
                % 省略之前的列表内容
    
                \noindent 下面是一个多层次列表的示例：
                \begin{enumerate}
                    \item 第一个选项
                    \item 第二个选项
                    \item \begin{enumerate}
                        \item 第一个选项
                        \item 第二个选项
                        \item \begin{enumerate}
                            \item 第一个选项
                            \item 第二个选项
                            \item 第三个选项
                        \end{enumerate}
                    \end{enumerate}
                \end{enumerate}
        %... 省略后续内容
    \end{document}
    

如你所见，我们在上述示例中选套定义了四个`enumerate`环境，分别对应三层嵌套的多层次列表。在默认情况下，\\(\\LaTeX\\) 会为每个列表项自动添加编号，并按照嵌套的层级关系递增。

#### 注释说明

在写作过程中，我们有时还会需要对特定的内容添加一些注释说明，以帮助读者更好地理解文档内容。在 \\(\\LaTeX\\) 语言中，我们可以通过`\footnote{}`命令来设置脚注，或者通过`\marginpar{}`命令来设置边注。例如，如果我们想在上述文章中的“注释说明”这个二级标题下面分别添加一段被脚注和边注的文本，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    \marginparwidth=2cm  % 设置边注的宽度
    \marginparsep=0.2cm  % 设置边注与正文的间距
    
    % 开始组织文档内容
    \begin{document}
        \section{正文样式示例}
            % 省略之前的章节
            \subsection{注释说明}
                \noindent 这是一段要被脚注的文本。
                \footnote{这是脚注的内容。}
    
                \noindent 这是一段要被边注的文本。
                \marginpar{这是右侧边注内容。}
        %... 省略后续内容
    \end{document}
    

在上述示例中，我们首先使用`\footnote{}`命令设置了一段脚注内容。在脚注内部，我们可以使用普通的文本命令（如`\textit{}`、`\underline{}`等）来设置这段文本的样式。接下来，我们使用`\marginpar{}`命令设置了一段边注内容。在边注内部，我们同样可以使用普通的文本命令来设置这段文本的样式。需要注意的是，在设置边注之前，我们需要先在文档的全局设置区使用`\marginparwidth`命令来设置边注的宽度，并通过`\marginparsep`命令来设置边注与正文的间距。

现在，如果我们再次编译这个文档，就可以看到“注释说明”这个二级标题下面已经出现了上面所设置的脚注和边注，如图8所示：

需要提醒读者的是，在加载了`ctex`宏包的中文文档中，`\marginpar`命令所设置边注的具体位置可能受版面布局影响而出现偏移；如需更稳定的边注布局，使用`marginnote`宏包会个更好的选择。关于这方面的操作，读者可参照`marginnote`宏包的文档说明来进行。我们在这里就不再继续展开了。

标题页与目录
------

在 \\(\\LaTeX\\) 排版系统中，完成针对文档内容的排版后，我们通常会为其添加一个标题页（也可以理解为封面），并根据它的章节层级为其自动生成一个目录。其中，用于生成标题页的命令为`\maketitle`，而生成目录的命令则是`\tableofcontents`。例如，如果我们想在上述文章中添加一个标题页和一个目录，就可以像下面这样写：

    % 省略之前的文档结构定义部分
    % 开始组织文档内容
    \begin{document}
        \maketitle        % 添加标题页
        \tableofcontents  % 添加目录
        \newpage          % 添加分页符
        \section{正文样式示例}
            % 省略之前的章节
            \subsection{注释说明}
                % 省略之前的注释说明
        %... 省略后续内容
    \end{document}
    

如你所见，我们在`\begin{document}`命令之后，章节层级的定义之前的位置上添加了三条命令。首先，`\maketitle`命令用于自动根据我们之前设置的文档元信息来生成标题页。接下来，`\tableofcontents`命令则会根据`\section{}`、`\subsection{}`等命令所定义的章节层级来自动生成文档的目录。最后，`\newpage`命令用于在目录与正文之间添加一个分页符，以便让正文内容从新的一页开始。

另外在默认情况下，目录所涉及的章节层级只有三级，如果读者想涉及到更多或更少的章节层级，就需要在文档的全局设置区使用`\setcounter{tocdepth}{层级数}`命令来设置目录的层级深度，例如像下面这样：

    % 省略之前的文档结构定义部分
    \setcounter{tocdepth}{2}  % 设置目录的层级深度
    
    % 开始组织文档内容
    \begin{document}
        \maketitle  % 添加标题页
        \tableofcontents  % 添加目录
        \section{正文样式示例}
            % 省略之前的章节
            \subsection{注释说明}
                % 省略之前的注释说明
        %... 省略后续内容
    \end{document}
    

在上述示例中，我们使用`\setcounter{tocdepth}{4}`命令将目录的层级深度设置为4级。现在，如果我们再次编译这个文档，就可以看到“正文样式示例”这个一级标题下面已经出现了上面所设置的标题页和目录，如图9所示：

笔记小结
----

在本篇笔记中，我们主要介绍了 \\(\\LaTeX\\) 语言中的一些基本排版功能，包括文档的排版结构、标题样式、正文样式、列表元素、注释说明、标题页与目录。通过这些排版功能，我们可以方便地设置出一篇纯文本文档的排版结构、样式以及内容。当然，针对更为专业的科普、学术文档，\\(\\LaTeX\\) 系统还提供了许多其他高级排版功能，如表格、图片、公式等。在后续的笔记中，我们将继续介绍这些高级排版功能。
