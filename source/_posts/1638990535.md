【译】Visual Studio 17.10 发布了新版扩展管理器
=================================

从 Visual Studio 17.10 开始提供新的扩展管理器作为默认预览功能。我们已将基本功能简化为现代风格 UI，以帮助您发现新的扩展并管理已安装的扩展。

　　我们将更新的扩展管理器带给所有用户！在过去的一年里，我们已经将更新后的扩展管理器作为可选的预览功能提供，并一直期待您的反馈。基于您令人难以置信的反馈，我们现在准备从 Visual Studio 17.10 开始提供新的扩展管理器作为默认预览功能。我们已将基本功能简化为现代风格 UI，以帮助您发现新的扩展并管理已安装的扩展。

　　有了这个版本，我们仍然想听到您的声音！当您在最后一个预览阶段使用扩展管理器时，请分享您的想法，以便我们可以在即将发布的正式版本中为您带来最好的体验。

**来看看最新的扩展管理器**
===============

　　离开 Visual Studio 去寻找有关扩展的更多信息的日子已经一去不复返了！我们创建了新的扩展管理器，以便在搜索扩展时突出显示最重要的内容，包括一个用于查看详细描述的大窗口。

![](https://img2024.cnblogs.com/blog/270073/202406/270073-20240610150939103-1352965842.png)

　　左边栏可以让您选择主视图：

　　- 浏览 Visual Studio Marketplace 以获取新的扩展

　　- 查看和管理已安装的扩展

　　- 发现和安装任何可用的更新为您的扩展

　　- 安装启用漫游的任何扩展

![](https://img2024.cnblogs.com/blog/270073/202406/270073-20240610151002636-1058747409.png)

**过滤搜索**
========

　　您可以单击搜索框右侧的过滤图标，显示类别和子类别筛选，您可以使用这些下拉列表过滤搜索结果。例如，您可以搜索帮助处理文档的工具或 ASP. NET 控件。

![](https://img2024.cnblogs.com/blog/270073/202406/270073-20240610151022358-838405644.png)

**让我们知道您的想法！**
==============

　　新的扩展管理器对所有 Visual Studio 17.10 用户都可用，并且是默认开启的。

　　如果出于任何原因您想要返回到以前的扩展管理器，您可以通过 Tools > Manage Preview Features 并取消选中名为 Extension Manager UI Refresh 功能的框来禁用预览功能。要重新启用新的扩展管理器，请选中预览功能的复选框并重新启动 Visual Studio。

　　当您使用新的扩展管理器时，请在开发人员社区分享您的想法和反馈。我们希望听到您关于任何可能缺失或可以改进的信息。

　　如果您遇到任何错误或问题，请告诉我们，以便我们可以优先修复它们。
