.NET 6.0.6 和 .NET Core 3.1.26、Visual Studio 2022 17.2 和 17.3 Preview 2 和 .NET 7.0 Preview 5 同时发布
================================================================================================

Microsoft 昨天发布了适用于 .NET 6.0.6 和 .NET Core 3.1.26、NuGet、Visual Studio 2019 和 Visual Studio 2022 17.2 的安全更新，其中更正了 [CVE-2022-30184](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2022-30184)。

在Visual Studio 2022的新版本17.2中，进行了进一步的更正，您可以[在此处阅读](https://docs.microsoft.com/en-us/visualstudio/releases/2022/release-notes#17.2.4)。除了 .NET 6.0.6 和 .NET Core 3.1.26 之外，还有 .NET 7.0 的新预览版 5。但是，此处不包括此漏洞。因此，它“仅”是对新版本的更新。

Visual Studio 2022 17.3 Preview 2也于昨天发布。此版本现在包括 Windows 11 上的本机 ARM 支持。

_“此预览版包括 Windows 11 上的本机 Arm64 支持！这将是Visual Studio的第一个本机支持在基于Arm的处理器上构建和调试Arm64应用程序的版本。我们将支持以下方面：”_

*   使用C++进行桌面开发（适用于基于 MSBuild 的项目）
*   .NET Desktop Development （WinForms， WPF）
*   ASP.NET 和网站开发
