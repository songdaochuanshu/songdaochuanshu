真香警告！JitPack 开源库集成平台
====================

请各大网友尊重本人原创知识分享，谨记本人博客：[南国以南i](https://home.cnblogs.com/u/bgyb/)

简介
==

_**官方介绍：**_

**[JitPack](https://jitpack.io/) 是一个用于 JVM 和 Android 项目的新颖的包存储库**。它按需构建 Git 项目并为您提供即用型工件（jar、aar）。

如果您希望**您的开源库可供全世界使用**，则无需执行项目构建和上传步骤。您需要做的就是将您的项目推送到 GitHub，然后 JitPack 会处理剩下的事情。真的是这样！

如果您的项目已经在 GitHub 上，JitPack 确保它可以由任何人构建。想在您的项目中使用 GitHub 库吗？

_**本人理解：**_

**项目开发中会用到很多开源库,他们一般通过Maven/Gradle依赖进来的**。[**JitPack**](https://jitpack.io/) 像是一个自定义的**Maven仓库**，输入**Github或Gitee**仓库地址(或其他代码托管平台),快速发布项目,获取不同版本的开源库依赖,快速被引用

使用场景:
=====

1.  本地没有私服,又可以和远程的小伙伴共同使用同一的代码
2.   方便了开源项目,可以更快的被流传使用

**官方Git Hub ：**[https://github.com/jitpack/jitpack.io](https://github.com/jitpack/jitpack.io)

**JitPack 官方构建示例：**[https://docs.jitpack.io/#snapshots](https://docs.jitpack.io/#snapshots)

一、准备工作
======

1.1 生成私人令牌
----------

因为 JitPack 需要授权才能访问 Gitee 中的代码，所以先在 Gitee 中生成一个私人令牌。在 Gitee 个人设置中选择私人令牌，然后生成私人令牌。  
`友情提示：`将**私人令牌串码保存到记事本中**，一会在 JitPack 中要用。

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506200513556-1357470594.png)

1.2. 在[JitPack](https://jitpack.io/ "JitPack")中配置 Gitee 访问权限
------------------------------------------------------------

先用你的Github账号登录JitPack

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506200728614-1325028122.png)

二、准备Gitee 码云项目
==============

2.1 将本地的开发项目上传到 Gitee仓库
-----------------------

`友情链接：`参考笔者之前写的一篇文章：[本地项目推送到远程仓库（原来可以这么玩）](https://www.cnblogs.com/bgyb/p/15243261.html)

2.2 为源码仓库创建 发行（Release）版本
-------------------------

**1\. 在 Git 添加标签 tag，并且提交到 Gitee**

git tag -a <tagname> -m <msg>
git push \--tags

**2\. 在 Gitee项目中创建发行版本**

在 Gitee 的项目中点击点击 `"创建发行版"`按钮

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506201610638-1575548268.png)

填写必要的信息，即可创建一个新的 发行版本：

 ![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506201803151-940979776.png)

三、开源库集成JitPack
==============

3.1 进入 [JitPack](https://jitpack.io/)，使用 [GitHub](https://github.com/) 账号登录
---------------------------------------------------------------------------

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220508161602161-517602944.png)

3.2  JitPack 编译开源库
------------------

当项目代码上传到`Gitee`创建发行版后，点击`JitPack`按的 `"Look Up"`按钮会需要等待一段时间后才会有版本的显示，因为`JitPack`要从`Gitee`上获取发行版代码zip文件，然后解压编译，所以请耐心等待数十秒，可自动进行发布

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506203540084-1504149483.png)

**查看编译日志：**点击 `"Log"` 的图标可以查看 `"build"`  的日志。

 ![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506203029619-1546865458.png)

**查看引入依赖示例：**点击`Get it` 的图标可以查看项目引用依赖配置说明

 ![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220506203935084-853108203.png)

四、本地项目引用开源库
===========

4.1 解锁Jit Pack 编译的项目
--------------------

点击设置按钮，如果是锁定状态，远程无法使用和访问下载依赖，需要点击解锁，如2状态则可以远程访问。

![](https://img2022.cnblogs.com/blog/1867541/202205/1867541-20220508150636256-728974296.png)

4.2 本地项目的pom.xml文件引入两个依赖
------------------------
