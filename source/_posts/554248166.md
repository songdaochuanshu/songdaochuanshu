PHP 程序员学会了 Go 语言就能唬住面试官吗？
=========================

![PHP 程序员学会了 Go 语言就能唬住面试官吗？](https://img2024.cnblogs.com/blog/737276/202408/737276-20240819231142040-1997667242.png) 唬住了 50k ，唬不住就 5k 。这句话一直是 PHP 程序员之间相互吹捧、吹牛逼的笑点，每次面试过后都会挠挠头上仅剩的几根头发，回想自己是否吹牛逼会过了头。

大家好，我是码农先森。

唬住了 50k ，唬不住就 5k 。这句话一直是 PHP 程序员之间相互吹捧、吹牛逼的笑点，每次面试过后都会挠挠头上仅剩的几根头发，回想自己是否吹牛逼会过了头。我经常在微信程序员撕逼群里，看到群友因为某个技术点而争执不休，时而讨论这个技术能不能在面试过程中突出自己的水平，从而拉高档次砍到高工资，但是又怕自己吹的技术漏出破绽，使自己在这场面试中尴尬的黯然离场。面试其实是一门技术活，有些人技术水平高但又谈不到高工资，反而有部分人技术水平不咋地工资却确挺高的，你说气不气人，况且这种现象还不少。

![](https://img2024.cnblogs.com/blog/737276/202408/737276-20240819230528710-1064015924.png)

大家应该都有不少的面试经历，面对不同的面试官，需要使出各种各样招数和技能，遇到\*\*\*钻一点的面试官，还得使出祖传的杀手锏，如果这一招也失效了，那只能灰溜溜的出门左转了。面试就像是升级打怪一般，过了一轮又一轮，就算你勇敢的闯到了最后一轮，还要面临 HR 对薪资的打压，这简直是心痛不已啊。记得我有一次职业生涯中最刻苦铭心的面试经历，每一轮面试两个面试官，一天面了四轮，前前后后八个面试官参与，心态都要爆炸，最后一轮 HR 对我说你很符合我们的用人需求，已经进入了我司的候选人名单了，回去等通知吧。于是我兴高采烈的回去了，我心想搞了这么多轮面试，这个 Offer 应该是拿捏的死死的了吧，等了一天又一天，一周又一周，时间过了许久，我都已入职其他的公司，也没等来通知。初入职场了我，就这么被狠狠地上了一课，认识到了这个职场社会的人心难测。

![](https://img2024.cnblogs.com/blog/737276/202408/737276-20240819230545343-516162884.webp)

说了这么多，还是回归下我们的主题「PHP 程序员学会了 Go 语言就能唬住面试官吗」。其实能不能唬住，这个也得分情况，如果本身这个面试官技术水平也很菜，能唬住也很正常，要是面试官技术很好，那大概率会被吊打。这一点我也有一段亲身经历，在面试某家公司时，首轮面试我的是一个普通的 PHP 程序员，看我简历上写了熟悉使用 Go 语言，就开始一本正经的问我 Go 语言相关的问题，首个问题是 Go 语言中的协程和 Swoole 中协程的区别，经过我一顿的疯狂输出，把他震惊了。于是顺利的来到了第二轮面试，我正襟危坐的在面试办公室等着，这时一位留着地中海发型油腻的架构师走了进来，他接二连三的问了一些 PHP 语言和 Go 语言的底层知识、实现原理、设计思想等，结果可想而知，被一顿吊打，简直把我干懵逼了。

![](https://img2024.cnblogs.com/blog/737276/202408/737276-20240819231028293-2140139092.jpg)

能熟练的使用 Go 语言在 PHP 程序员的面试过程中确实是一大亮点，但也仅限在简历筛选和首轮面试的过程中会展现出明显的优势，随着面试的深入很多的缺陷就会逐渐的暴露出来。如果技术吹过了头，反而会得不偿失，不利于后续的面试进程，所以劝诫 PHP 靓仔们吹牛也要适度，切勿贪图吹牛的一时爽快，避免出现「面试一时爽，结果火葬场」的惨象。总而言之，唬住面试官这个是概率事件，打铁还需自身硬，要用自己的真实技术水平来镇住面试官，而不能夸夸其谈，也不能总是抱着唬住一个算一个的伎俩，这样的招数终究不是长久之计。本次分享的内容就到这里了，希望对大家能所有启发。
